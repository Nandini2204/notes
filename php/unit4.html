4.1 Configuring PHP For Database Support:-जब हम mysqli Extension को Object Oriented तरीके से उपयोग में लेना चाहते हैं, तो हमें mysqli Class को उसके Constructor के आधार पर Instantiate करना होता है। इसका Constructor निम्नानुसार होता हैः mysqli(string host, string username, string pwd, string db_name, int port, string socket)
इस Constructor में MySQL Database के Host जो कि सामान्यतः localhost होता है, Database के Username, Password, Connect किए जाने वाले Database के db_name, MySQL के Connecting Port व Socket आदि Information को Specify करके हम PHP का Connection MySQL के साथ करते हैं। ये सभी Parameters लगभग वही हैं, जिन्हें हमने पिछले Section में mysqli_connect() Function में Specify किया है।
नया Connection बनाने के लिए यदि हम Object Oriented तरीके को Use करना चाहें, तो हम ये काम निम्नानुसार कर सकते हैं, जैसाकि हमने पहले mysqli_connect() Function को Use करते हुए किया थाः $db_conn = new mysqli(“localhost”, “registration”, “pwd”, ” registration”);


हालांकि हम इस Constructor में ही उस Database को Specify कर सकते हैं, जिसे हम Access करना चाहते हैं, लेकिन mysqli हमें connect() व select_db() नाम के दो Methods Provide करता है, जिनका प्रयोग करके हम हमारी जरूरत के अनुसार अपनी PHP Script में कभी भी कहीं भी नए Database Select कर सकते हैं और नए Database से PHP को Connect कर सकते हैं।
connect() Method वही Parameters Accept करता है, जो पिछला Constructor Accept कर रहा है। जैसेः
// Instantiate the mysqli class
$db_conn = new mysqli();
// Connect to the database server and select a database
$db_conn->connect(“localhost”, “registration”, “pwd”, ” registration”);
Connection बनाने के बाद Database Select करने के लिए हम select_db() Method को निम्नानुसार Use कर सकते हैं:
// Connect to the database server
$db_conn = new mysqli();
$db_conn->connect(“localhost”, “registration”, “pwd”);
// Select the database
$db_conn->select_db(“registration”);
PHP का MySQL के साथ Connection Establish करने व उपयुक्त Database को Select कर लेने के बाद हम उस Connection पर किसी भी प्रकार की Database SQL Queries जैसे कि Inserting, Updating, Selecting, Deleting आदि को Fire या Execute कर सकते हैं।
जब एक बार PHP Script Finish हो जाती है, तो उस PHP Script में Open किए गए सभी Database Connections Automatically Close हो जाते हैं और Database द्वारा Occupied सभी Resources Free हो जाते हैं। फिर भी क्योंकि हम एक ही PHP Script में एक से ज्यादा Connection Open कर सकते हैं, इसलिए यदि हम चाहें तो हम हमारी जरूरत के अनुसार PHP Script में कहीं भी कभी भी किसी भी Database Connection को close()Method का प्रयोग करके Close कर सकते हैं। जैसेः
// Connect to the database server
$db_conn = new mysqli();
$db_conn->connect(“localhost”, “registration”, “pwd”);
// Select the database
$db_conn->select_db(“registration”);
// Interaction code with Database.
$db_conn->close();
कई बार ऐसी स्थितियां भी होती हैं, जब हम जिस MySQL Database से Connect होना चाहते हैं, किसी कारणवश उस Database से Connection नहीं हो पाता। परिणामस्वरूप Error Trigger होता है। इन Connection Related Errors को Handle करने के लिए mysqli Extension हमें mysqli_connect_errno() व mysqli_connect_error() नाम के दो Functions Provide करता है। Error Handling के लिए हम निम्नानुसार PHP Code लिख सकते हैं: < ?php
// Connect to the database server
$db_conn = new mysqli(“localhost”, “registration”, “pwd”, ” registration”);
if ($db_conn->errno) {
printf(“Unable to connect to the database:
%s”, $db_conn->error);
exit();
}
?>


4.2 PHP’s Database API’s:- एक application programming interface या API, वर्गों, विधियों, कार्यों और चर को परिभाषित करता है जिन्हें आपके आवेदन को अपने वांछित कार्य को करने के लिए कॉल करना होगा। PHP अनुप्रयोगों के मामले में, जिन्हें डेटाबेस के साथ संचार करने की आवश्यकता होती है, आमतौर पर PHP एक्सटेंशन के माध्यम से आवश्यक एपीआई को उजागर किया जाता है।
एपीआई प्रक्रियात्मक या वस्तु-उन्मुख हो सकते हैं। एक प्रक्रियात्मक एपीआई के साथ आप कार्यों को पूरा करने के लिए फ़ंक्शन कहते हैं, ऑब्जेक्ट-ओरिएंटेड एपीआई के साथ आप कक्षाओं को तत्काल करते हैं और फिर परिणामी वस्तुओं पर तरीकों को कॉल करते हैं। दो में से उत्तरार्द्ध आमतौर पर पसंदीदा इंटरफ़ेस है, क्योंकि यह अधिक आधुनिक है और बेहतर संगठित कोड की ओर जाता है।
जब PHP अनुप्रयोगों को MySQL सर्वर से कनेक्ट करने की आवश्यकता होती है, तो कई एपीआई विकल्प उपलब्ध होते हैं। यह दस्तावेज़ चर्चा करता है कि क्या उपलब्ध है और अपने आवेदन के लिए सबसे अच्छा समाधान का चयन कैसे करें।


4.3 PHP’s SQL API:-PHP एक server side, HTML- embedded scripting language है जिसका उपयोग dynamic web पेज बनाने के लिए किया जा सकता है। यह अधिकांश ऑपरेटिंग सिस्टम और वेब सर्वर के लिए उपलब्ध है, और MySQL सहित अधिकांश सामान्य database तक पहुंच सकता है। PHP को एक अलग प्रोग्राम के रूप में चलाया जा सकता है या वेब सर्वर के साथ उपयोग के लिए एक module के रूप में संकलित किया जा सकता है।
PHP चार अलग-अलग MySQL API extention प्रदान करता है:
MySQL improved extention: “MySQL,improved” के लिए खड़ा है; यह extention PHP 5.0.0 के रूप में उपलब्ध है। यह MySQL 4.1.1 और बाद के उपयोग के लिए अभिप्रेत है। यह एक्सटेंशन MySQL 5.0 में उपयोग किए गए प्रमाणीकरण प्रोटोकॉल के साथ-साथ तैयार स्टेटमेंट और मल्टीपल स्टेटमेंट एपीआई का पूरी तरह से समर्थन करता है। इसके अलावा, यह एक्सटेंशन एक उन्नत, ऑब्जेक्ट-ओरिएंटेड प्रोग्रामिंग इंटरफ़ेस प्रदान करता है।
MySQL फ़ंक्शंस (PDO_MYSQL): इसका अपना API नहीं है, बल्कि यह PHP डेटाबेस एब्स्ट्रक्शन लेयर PDO (PHP डेटा ऑब्जेक्ट्स) के लिए MySQL ड्राइवर है। PDO MySQL ड्राइवर, PDO के नीचे की परत में बैठता है, और MySQL को विशिष्ट कार्यक्षमता प्रदान करता है। यह एक्सटेंशन PHP 5.1.0 के रूप में उपलब्ध है।


, Mysql_xdevapi: यह एक्सटेंशन MySQL के X DevAPI का उपयोग करता है और mysql_xdevapi नाम के PECL एक्सटेंशन के रूप में उपलब्ध है। सामान्य अवधारणाओं और X DevAPI उपयोग विवरण के लिए, X DevAPI उपयोगकर्ता मार्ग दर्शिका देखें।
मूल MySQL API: PHP संस्करण 4 और 5 के लिए उपलब्ध है, यह एक्सटेंशन MySQL 4.1 के पहले MySQL संस्करणों के साथ उपयोग के लिए है। यह एक्सटेंशन MySQL 4.1 में उपयोग किए गए बेहतर प्रमाणीकरण प्रोटोकॉल का समर्थन नहीं करता है, न ही यह तैयार किए गए बयानों या कई बयानों का समर्थन करता है। MySQL 4.1 के साथ इस एक्सटेंशन का उपयोग करने के लिए, आप पुराने_पासवर्ड सिस्टम चर को 1 पर सेट करने के लिए MySQL सर्वर को कॉन्फ़िगर करेंगे (देखें कि क्लाइंट प्रमाणीकरण प्रोटोकॉल का समर्थन नहीं करता है)।

4.5 MYSQL V/S SQL Server:-मायएसक्यूएल और एसक्यूएल सर्वर के बीच अंतर पर चर्चा करें:
• MYSQL मुख्य रूप से पीएचपी परियोजनाओं या अनुप्रयोगों के लिए उपयोग किया जाता है।SQL Server मुख्य रूप से .नेट अनुप्रयोगों या विंडोज प्रोजेक्ट्स के लिए उपयोग किया जाता है।
• MYSQL में, कई स्टोरेज इंजन हैं जो प्रदर्शन के आधार पर टेबल के लिए इंजन का उपयोग करने के लिए विकास को अधिक लचीलापन देते हैं।लोकप्रिय स्टोरेज इंजनों में से एक ईन्नोडीबी है। SQL Server केवल एक या एक स्टोरेज इंजन का उपयोग कर सकते हैं।
• MYSQL में, एक बार क्वेरी निष्पादित हो जाने के बाद, इसे मध्य-मार्ग में रद्द नहीं किया जा सकता है।एसक्यूएल सर्वर में, निष्पादन के बाद क्वेरी को मध्य-मार्ग में रद्द किया जा सकता है।
• MYSQL में कोई उपकरण नहीं है जो सुरक्षा प्रदान करता है।एसक्यूएल सर्वर में सुरक्षा उपकरण है जो डेटा सुरक्षा इत्यादि सुनिश्चित करता है जो एमएस बेसलाइन सुरक्षा विश्लेषक उपकरण है।
• MYSQL निःशुल्क और उपयोग करने के लिए स्वतंत्र है।SQL Server लाइसेंस प्राप्त है और एकाधिक डेटाबेस का उपयोग करने और चलाने के लिए लाइसेंस खरीदने की आवश्यकता है। कई लाइसेंस खरीदने के लिए वास्तव में महंगा है।
• MYSQL के लिए, समर्थन या सहायता प्राप्त करने के लिए भुगतान करना आवश्यक है।SQL Server के पास व्यापक सामुदायिक समर्थन या ऑनलाइन मंच हैं। माइक्रोसॉफ्ट समर्थन भी बीएसक्यूएल सर्वर को समर्थन प्रदान करने के लिए भुगतान किया जाता है।
• यदि MYSQL .नेट के साथ उपयोग किया जाता है, तो प्रारंभ करने के लिए तीसरे पक्ष के उपकरण डाउनलोड करने की आवश्यकता होती है।यदि SQL Server का उपयोग .नेट के साथ किया जाता है, इनबिल्ट फ्रेमवर्क कक्षाएं संगतता प्रदान करती हैं और क्वेरीज़ के साथ भी शुरू की जा सकती हैं।
• MYSQL एंटरप्राइज़ मैनेजर का उपयोग आईडीई उपकरण के रूप में करता है।क्वेरी सर्वर विकसित करने के लिए SQL Server प्रबंधन स्टूडियो का उपयोग आईडीई के रूप में करता है।


4.6 Database Drivers:-एक कंप्यूटर सिस्टम में, विभिन्न प्रकार के किसी अन्य सिस्टम से कनेक्शन बनाने के लिए एक एडेप्टर प्रोग्राम की आवश्यकता होती है। प्रिंटर ड्राइवर का उपयोग करके कंप्यूटर से प्रिंटर कनेक्ट करने के समान, एक DBMS (डेटाबेस मैनेजमेंट सिस्टम) को एक डेटाबेस ड्राइवर की आवश्यकता होती है जो अन्य सिस्टम में डेटाबेस कनेक्शन को सक्षम करता है।
एक डेटाबेस ड्राइवर एक कंप्यूटर प्रोग्राम है जो एक डेटाबेस कनेक्शन के लिए एक प्रोटोकॉल (ODBC या JDBC) को लागू करता है।
ड्राइवर एक एडाप्टर की तरह काम करता है जो एक विशिष्ट डेटाबेस विक्रेता कार्यान्वयन के लिए एक सामान्य इंटरफ़ेस को जोड़ता है। जावा एप्लिकेशन में डेटाबेस कनेक्शन बनाने के लिए, हमें JDBC (जावा डेटाबेस कनेक्टिविटी एपीआई) ड्राइवर की आवश्यकता होती है। व्यक्तिगत डेटाबेस से जुड़ने के लिए, JDBC को प्रत्येक विशिष्ट डेटाबेस प्रकार के लिए ड्राइवरों की आवश्यकता होती है।
उदाहरण के लिए, यदि किसी जावा एप्लिकेशन को MySQL डेटाबेस से डेटा स्रोत के लिए कनेक्शन की आवश्यकता होती है, तो हमें लाइब्रेरी MySQL JDBC ड्राइवर को शामिल करना होगा जो कि एक जार फ़ाइल है, अर्थात् mysql-कनेक्टर-जावा-5.1.23-bin.jar। जार फ़ाइल में संस्करण संख्या भिन्न हो सकती है। ड्राइवर का नाम com.mysql.jdbc.Driver है|
