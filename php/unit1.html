1.2 Dynamic Contents From Database:-

आपके webpage पर कोई भी content जो परिवर्तन के अधीन है, को dynamic content के रूप में devide किया गया है। ऐसी content तब तक आउट-डेटेड हो जाती है जब तक कि उसे अक्सर अपडेट न किया जाए। उदाहरण के लिए, हर बार जब आप एक नया product लॉन्च करते हैं, तो इसे product Portfolio page पर जोड़ा जाना चाहिए। या हर बार जब कोई client आपके product के बारे में कुछ अच्छा कहता है, तो उसे client प्रशंसापत्र पृष्ठ (Testimonial page) पर शामिल करने की आवश्यकता होती है। HTML से निपटने के लिए बिना अपनी content को Up to date रखने के लिए डायनामिक content सबसे आसान तरीका है। डायनेमिक कंटेंट Zoho Creator डेटाबेस द्वारा संचालित है। आपकी सभी webpage content डेटाबेस पर संग्रहीत है, और डेटाबेस आपके वेबपेज से जुड़ा हुआ है। हर बार content को डेटाबेस में store किया जाता है, परिवर्तन तुरंत वेबपेज पर अपडेट किए जाते हैं। डायनामिक content को कैसे configure करें:
1. अपने page के उस क्षेत्र में “डायनामिक कंटेंट” विजेट को ड्रैग करें जहां आप कंटेंट को बार-बार बदलना चाहते हैं।
2. आपके सभी डेटाबेस की सूची के साथ एक डायलॉग बॉक्स दिखाई देता है। उपयुक्त एप्लिकेशन और उस एप्लिकेशन के भीतर डेटाबेस चुनें जो आपके डेटा को रखता है। आगे बढ़ने के लिए OK पर क्लिक कर
ें 3. अगला, आपको dynamic content के layout को कॉन्फ़िगर करने की आवश्यकता है; आपके webpage पर डेटा की व्यवस्था कैसे की जानी चाहिए। layout page में 9 सेल हैं, जिसमें आप left side से फ़ील्ड को खींच सकते हैं। आप अपनी content को अपने प्रकाशित webpage पर display करने के लिए अपनी content को Aligned करने के लिए मर्ज या विभाजित कर सकते हैं।
4. एक बार जब आप Desire लेआउट कॉन्फ़िगर कर लेते हैं, Dynamic content के पीछे डेटाबेस Zoho निर्माता द्वारा संचालित हैं।


1.3 Developing Dynamic Internet Applications:-

सर्वर-साइड डायनामिक वेब पेज एक वेब पेज है जिसका निर्माण एक एप्लिकेशन सर्वर प्रोसेसिंग सर्वर-साइड स्क्रिप्ट द्वारा नियंत्रित किया जाता है। सर्वर-साइड स्क्रिप्टिंग में, पैरामीटर निर्धारित करते हैं कि प्रत्येक नए वेब पेज की असेंबली कैसे और अधिक क्लाइंट-साइड प्रोसेसिंग की स्थापना सहित।

क्लाइंट-साइड डायनामिक वेब पेज लोड होते ही ब्राउज़र में चल रहे HTML स्क्रिप्टिंग का उपयोग करके वेब पेज को प्रोसेस करता है। जावास्क्रिप्ट और अन्य स्क्रिप्टिंग भाषाएं निर्धारित पृष्ठ में HTML को document object model(DOM) में pass किए गए तरीके से निर्धारित करती हैं, जो लोड किए गए वेब पेज का प्रतिनिधित्व करता है। उसी क्लाइंट-साइड तकनीकों को उसी तरह DOM को डायनामिक रूप से update या परिवर्तित किया जा सकता है। एक dynamic वेब पेज को तब user द्वारा या कुछ variable containt को बदलने के लिए कंप्यूटर प्रोग्राम द्वारा पुनः लोड किया जाता है। update information सर्वर से, या उस page के DOM में किए गए परिवर्तनों से आ सकती है। यह ब्राउज़िंग history को काट नहीं सकता है या वापस जाने के लिए सहेजा गया version नहीं बना सकता है, लेकिन Ajax technoilogies का उपयोग करके एक dynamic वेब पेज अपडेट न तो वापस जाने के लिए एक पेज बनाएगा, और न ही प्रदर्शित पृष्ठ के आगे वेब ब्राउज़िंग इतिहास को छोटा करेगा। अजाक्स technologies के उपयोग से अंतिम user को वेब ब्राउज़र में एक page के रूप में प्रबंधित एक dynamic page मिलता है जबकि उस page पर prodev की गई real वेब containt भिन्न हो सकती है। Ajax इंजन केवल अपने क्लाइंट के लिए अपने DOM, DOM के हिस्सों के लिए, अपने क्लाइंट के लिए एक एप्लीकेशन सर्वर से बैठता है।

1.4 Client Side Scripting versus Server Side Scripting

Difference between Server-side Scripting and Client-side Scripting:-

1. सर्वर साइड स्क्रिप्टिंग का उपयोग डायनेमिक पेज बनाने के लिए किया जाता है, जब उपयोगकर्ता ब्राउज़र सर्वर से अनुरोध करते हैं तो कई स्थितियां होती हैं।
2. क्लाइंट साइड स्क्रिप्टिंग का उपयोग तब किया जाता है जब उपयोगकर्ता ब्राउज़र में पहले से ही सभी कोड होते हैं और पृष्ठ को उपयोगकर्ता इनपुट के आधार पर बदल दिया जाता है।
3. >> वेब सर्वर उस सर्वर साइड स्क्रिप्टिंग को निष्पादित करता है जो ब्राउज़र को भेजे जाने वाले पृष्ठ का उत्पादन करता है।


4. वेब ब्राउज़र क्लाइंट साइड स्क्रिप्टिंग को निष्पादित करता है जो उपयोगकर्ता के कंप्यूटर पर रहता है।
5. >> सर्वर एक पृष्ठ भेजने के लिए सर्वर-साइड स्क्रिप्ट निष्पादित करता है, लेकिन यह क्लाइंट-साइड स्क्रिप्ट निष्पादित नहीं करता है।
6. ब्राउज़र सर्वर द्वारा भेजे गए पृष्ठ को प्राप्त करता है और क्लाइंट-साइड स्क्रिप्ट निष्पादित करता है।
7. >> सर्वर साइड स्क्रिप्टिंग का उपयोग वेब सर्वर पर रहने वाले डेटाबेस से जुड़ने के लिए किया जाता है।
8. क्लाइंट साइड स्क्रिप्टिंग का उपयोग वेब सर्वर पर डेटाबेस से कनेक्ट करने के लिए नहीं किया जा सकता है।
9. >> सर्वर साइड स्क्रिप्टिंग वेब सर्वर पर रहने वाले फाइल सिस्टम तक पहुंच सकता है।
10. क्लाइंट साइड स्क्रिप्टिंग उस फ़ाइल सिस्टम तक नहीं पहुंच सकती जो वेब सर्वर पर रहता है। 11. >> सर्वर साइड स्क्रिप्टिंग भाषाओं के उदाहरण: PHP, JSP, ASP, ASP.Net, Ruby, Perl n कई और।
क्लाइंट साइड स्क्रिप्टिंग भाषाओं के उदाहरण: जावास्क्रिप्ट, वीबी स्क्रिप्ट आदि।


1.5 Advantage and Capabilities of PHP:-

Advantage of Php:-

1. PHP free और open source है यानि इसे आप फ्री में download करके उपयोग कर सकते हैं।
2. यह platform independent है यानि किसी भी platform जैसे Windows, Linux, Mac आदि में use किया जा सकता है।
3. इसका syntax बहुत आसान है इसलिए इसे बड़ी आसानी से सीखा जा सकता है।
4. इसका execution speed बहुत fast होता है।
5. Built-in database module का उपयोग कर बड़ी आसानी से database से connection create किया जा सकता है।
6. Powerful library support
7. नयी टेक्नोलॉजी और फीचर के साथ पीएचपी के versions लगातार update होते रहते हैं।
8. Apache और IIS दोनों तरह के servers के लिए compatible है।
9. PHP के साथ सिर्फ MySQL ही नही बल्कि अन्य प्रकार के डेटाबेस जैसे MS SQL Server, Oracle आदि भी उपयोग किये जा सकते हैं।
10. ज्यादातर hosting servers by default PHP support करते हैं। ASP की तरह इसके लिए dadicated hosting की जरुरत नही पड़ती। इसका मतलब यह है की PHP से बने website को होस्ट करने में अधिक पैसे खर्च नही करने पड़ते।

Capabilities of php:-

1.Dynamic website या web application बनाया जा सकता है।
2. वेबसाइट को डेटाबेस से connect कर सकते हैं।
3.PHP के जरिये database में data insert, update या delete किया जा सकता है।
4.User login system बनाया जा सकता है। server side validation भी हो सकता है।
5.PHP के जरिये आप यह तय कर सकते हैं की कौन सा user कौन से page को access कर सकता है।
6.Forms create कर सकते हैं जिसके जरिये user से data input करा कर database में store किया जा सकता है।
7.Email send और receive किया जा सकता है।
8.Browser के cookies को set और access किया जा सकता है।
9.Data encryption और decryption भी कर सकते हैं।


1.6 PHP vs ASP:-

PHP versus ASP के बीच के अंतर प्लेटफ़ॉर्म compatibility है। Linux, Unix, सोलारिस और विंडोज सहित कई विभिन्न प्लेटफार्मों पर काम करता है, जबकि ASP मुख्य रूप से विंडोज प्लेटफार्मों पर काम करता है। यदि आपके सर्वर पर ASP-Apache स्थापित है, तो आप लिनक्स पर ASP का उपयोग कर सकते हैं।
