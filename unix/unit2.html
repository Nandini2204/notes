-unix में लिखे जाने वाले प्रोग्राम्स इसी लिब डायरेक्टरी में स्टोर लाइब्रेरी फंक्शन्स का उपयोग करते है|

C)DEV:-

D)USER:-

इसमें अनेक डायरेक्टरी होती है व प्रत्येक डायरेक्टरी एक -एक यूजर से रिलेटेड होती है|
-वे यूजर डायरेक्टरी,सिस्टम एडमिनिस्ट्रेटर द्वारा उस समय क्रिएट की जाती है,जब वह many users के लिए अकाउंट क्रिएट करता है|
प्रत्येक यूजर अपनी डायरेक्टरी में वर्क करता है,जिसे उसकी होम डायरेक्टरी कहा जाता है और यह होम डायरेक्टरी में इच्छानुसार अनेक सब – डायरेक्टरी क्रिएट कर सकता है|
-यूजर डिरेक्टरी के अधीन एक बिन नामक डायरेक्टरी भीं होती है जो unix की कुछ enter command फाइल्स को स्टोर करके रखती है|

E)TEMP:-

डायरेक्टरी में unix यूजर द्वारा क्रिएट की गयी टेम्पररी फाइल्स को ही स्टोर किया जाता है|
-unix सिस्टम को सूत डाउन कर रीस्टार्ट करने पर ये टेम्पररी फाइल्स डिलीट हो जाती है|


2.3 File Permission:-unix एक secure operating system है।unix user की files और directories को unauthorized access से बचाने के लिए security provide करता है। unix permissions के माध्यम से files को security provide करता है। Permissions के माध्यम से unix को पता रहता है की कोई user किसी file के साथ क्या क्या कर सकता है।
Types of Linux Permissions:-
unix में 3 प्रकार की permissions पायी जाती है।
Read – जिन users के पास read permission होती है वह उस file को read कर सकते है।
Write – Write permission वाले users उस file में changes कर सकते है।
Execute – Execute permission वाले users उस file को execute कर सकते है।
Types of Linux User:-
Owner permissions:–Owner की permissions निर्धारित करती हैं कि फ़ाइल का owner फ़ाइल पर क्या कार्य कर सकता है।
Group:– एक users का group define करके भी उसके लिए permissions define की जा सकती है। उदाहरण के लिए आप चाहते है की कुछ users उस file को change न कर सके तो आप उन users को Write permission से हटा सकते है।
Others – इसमें पूरा world आ जाता है। आप एक ही बार में इसके द्वारा सबके लिए permissions define कर सकते है।
unix में permissions command line interface (terminal) से भी set की जा सकती है या फिर आप चाहे तो इसके लिए GUI (Graphical User Interface) भी use कर सकते है। किसी भी file से सम्बंधित user की permissions देखने के लिए निचे दी गयी command use की जा सकती है।
ls -l [file-path]
File path उस file का नाम होता है जिसकी permissions आप देखना चाहते है।
Changing Permissions:-फ़ाइल या directory permission को बदलने के लिए, आप chmod (परिवर्तन मोड) कमांड का उपयोग करते हैं। chmod का उपयोग करने के दो तरीके हैं – symbolic mode and the absolute mode.
unix chmod Command:-unix में chmod command का प्रयोग file या directories की permissions को change करने के लिए किया जाता है। इस command का पूरा नाम change mode होता है।
chmod permissions file-name :-सबसे पहले आप command define करते है। इसके बाद आप वे permissions define करते है जो आप उस file के लिए set करना चाहते है। इसके बाद file का नाम define किया जाता है। इस command को execute करते समय आप permissions को या तो number द्वारा या फिर symbol द्वारा represent कर सकते है। सभी संभव permissions के लिए numbers और symbols निचे table में दिए जा रहे है आप इनका प्रयोग करके जल्दी से permissions set कर सकते है।

एक बाद हमे हमेशा ध्यान रखनी चाहिए की permissions तीनों तरह के users (Owner, Group और Others) के लिए एक साथ set की जाती है। इसलिए जब भी हम numbers या symbol द्वारा permission set करते है तो 3 number या 3 symbol define करते है।
इनमे हर symbol हर type के user को संबोधित करता है। उदाहरण के लिए निचे दी गयी command को देखिये।
chmod 740 myFile
ऊपर दी command में 7 owner के लिए define किया गया है और बताता है की owner file को read, write और execute कर सकते है। इसी प्रकार 4 group के लिए है जो बताता है की एक special group के लोग सिर्फ image को देख सकते है और 0 बाकी सब के लिए जो बताता है की बाकी सब लोग के लिए इस file को देखने की भी permission नहीं है।

2.4 File Ownership:-हर unix सिस्टम में तीन प्रकार के मालिक होते हैं:

1.USER:-

एक user वह है जिसने फ़ाइल बनाई है। डिफ़ॉल्ट रूप से, जो भी बनाता है, फ़ाइल,फ़ाइल का owner बन जाता है। एक user फ़ाइल बना, हटा या modify कर सकता है।

2.GROUP:-

एक group में कई user हो सकते हैं। Group से संबंधित सभी user के पास फ़ाइल के लिए same access की permission है।

3.OTHER:-

कोई भी जिसके पास user और Group के अलावा फ़ाइल तक पहुंच है, other की category में आता है। other ने न तो फाइल बनाई है और न ही group मेंबर है। user और group स्थानीय रूप से / etc / psswd या / etc / group में manage किए जा सकते हैं।

unix chown: change owner:-Command chown is used to change the owner of the file.
Syntax:-chown 
Example:-

EXAMPLES

Change the owner of file.

$ chown user1 sample.txt

Change the group of file.

$ chown :mygroup file.txt

Change both the owner and group of file in single command.

$ chown user1:mygroup file.txt
कमांड chown का उपयोग user के owner और group दोनों को बदलने के लिए भी किया जा सकता है।
Syntax:-chown 
Example:-chown jtp:php msg.txt

2.6 Partition:-जब यूजर के फाइल ओर फोल्डर को आसानी से मैनेज करने के लिए हार्ड डिस्क को अलग पार्ट में devide करता है| तो हार्ड डिस्क के प्रत्येक पार्ट को partition कहा जाता है| हार्ड डिस्क को partition करके यूजर अपने डाटा को आसनी से manage कर सकता है |
Example:-मान लीजिये यूजर ने कंप्यूटर में हार्ड डिस्क के 4 partition बनाये है ओर प्रत्येक प्रकार में अलग अलग प्रकार के डाटा रखता है | जेसे अक partition में operating system ,दुसरे partition में सॉफ्टवेर व backup,तीसरे partition में song ओर movie और फोर्थ partition में पर्सनल document रखता है तो वह आसानी से अपने डाटा को manage कर सकता है |
Basic partition types of unix :-
1.Foreign Partiton 
2.Primary Partiton 
3.Logical Partition
4.Swap Partition
5.Unix Extended partion
6.Active Partion
1. Foregin Partion:-इस पर प्रकार के partition में अन्य operating system के फाइल system जेसे विंडो के NTFS ओर FAT फाइल system को consist करते है ओर इनके फीचर को उपयोग कर सकते है |

2. Primary partition:-यह partition Intel based System का ओरिजिनल partition है | इसक ज्यादातर उपयोग operating system को इनस्टॉल करने के लिए किया जाता है | unix operating system में maximum 4 प्राइमरी partition बनाये जा सकते है या हम 3 Primary Partition एवं एक extended partition बनाकर उन्हें अन्य लॉजिकल partition में devide कर सकते है

3. logical partition:-इस partition का उपयोग फाइल ओर फोल्डर को manage करने के लिए किया जाता है | इसे Primary partition का sub partition भी कहा जाता है | इस partition को हम active partition नहीं बना सकते है इसका तात्पर्य इस partition में operating system को इनस्टॉल नहीं किया जा सकता है |

4.Swap Partition:-यह partiton Windows operating system के वर्चुअल Memory की तरह कार्य करता है | वास्तव में swap partition hard disk के एक part होता है जो RAM की तरह कार्य करता है जब यूजर कंप्यूटर में कोई भी ऑपरेशन परफॉर्म करता है उस समय hard डिस्क से डाटा RAM के पास जाता है और जब यूजर multiple ऑपरेशन परफॉर्म करता है तब RAM की जगह कम पद जाती है उस टाइम operating system,हार्ड डिस्क के एक part को RAMकी तरह कार्य पर लेता है hard disk के उस जगह को ही swap partition कहा जाता है|


5.Active Partition:-Computer में hard डिस्क के जिस partition में operating system इनस्टॉल किया होता है उस partition को active partition कहा जाता है | ज्यादातर कंप्यूटर में 4 partition होते है जिसमे से एक partition में operating system को इनस्टॉल किया जाता है उस partition को ही active partition कहा जाता है |
