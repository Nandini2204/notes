4.1 Three modes:-Vi editor में three modes होते है:-

1.command mode
2.Input mode 
3.EX-mode 

1.COMMAND MODE:-

-vi editor का यह default mode होता है |
-इसमें press की गयी हर एक key एक command की तरह ही interpret होती है|
-उन्हें हम vi editor पर print होते नहीं देख सकते है|
-इस mode में हम copy,delete etc करने का कार्य कर सकते है|

2.INPUT MODE:-

-इस mode में press की गयी हर एक key display होती है|
-फाइल के कंटेंट भी होते है|
फुइले में कुछ write करने के लिए इस mode की आवश्यकता होतुई है|
इसे input text mode भी कहा जाता है|

3.EX-COMMAND MODE:-

इस mode vi command को command line से दिया जाता है -vi editor का यह mode फाइल handling mode होता है|
-इसमें फाइल को save आदि कार्यो का संपादन होता है|
command mode में इस mode में आने के लिए (:) colon का use किया जाता है|

4.2 INPUT MODE, ADDING AND REPLACING TEXT:-

किसी भी फाइल में text को इनपुट करने के लिए हमें vi के insert अर्थात इनपुट mode में जाना होगा|
-इनपुट mode में जाने के के लिए ‘i’ key press करनी पड़ती है|
-प्रत्येक line के ~(tilde) से स्टार्ट होती है|
हम जानते है की command mode से इनपुट mode में जाने के लिए verious keys को use करते है इनमे से कुछ निम्न है|
‘i’ :- यह text को insert करने के लिए सबसे simple type का command होता है|
-इसमें text current cursor position पर होता है|
“I”:- यह command current line के begining में text को insert करने के लिए use किया किया जाता है|
“a”:-इस command का use text को करंट cursor position के right hand side में apand करने के लिए होता है|
“A”:- यह command line के end में text को add करने के use में आता है|
“o”:-यह command line के नीचे न्यू line को ओपन करने के काम मे आता है|
“O”:-यह command current line के उपर न्यू line को ओपन करता है|
“r”:-यह command किसी single character को अन्य किसी single character द्वारा replace करने में use किया जाता है|
-यह command उस replacement को current cursor position पर ही apply करता है|
“R”:-यह command current cursor position के right हैण्ड वाली सभी text को replace करने में use लाया जाता है|
“s”:-यह command current single character position वाले अन्य character से replace करने में use लाया जाता है|
“S”:-यह command पूरी line को replace करने के काम में आता है|


4.3 Saving text and quitting – The ex mode:-EX mode फाइल से related output को पूरा करने के लिए होता है|
-फाइल में निम्न ऑपरेशन किये जा सकते है|
1.Saving text
2.Quiting text

1. SAVING TEXT:-

File को save करने के लिए ‘:w’ command का use किया जाता है|
-‘:w’ command फाइल को save करने के बाद control को प्रोग्राम editor/vi editor में ही रखता है 
“:x”:-यह command फाइल को save करने के बाद vi editor से भी बाहर निकाल देता है|
यह इनपुट command mode से बाहर निकल कर command mode पर ले आता है|
-दोनों ही उपरोक्त केस में file का name पहले से ही दिया हुआ होता है |
-यदि file का name पहले से नहीं दिया होता है तो :w command के साथ उस फाइल का name भी देते है जिसे save या exit भी करना है|
-उपरोक्त ‘:x’ command के साथ पर :w या command mode में ‘zz’ को भी फाइल save करने के लिए use में लायी जाती है|

2.QUITING TEXT:-

यह command फाइल को without save किये ही quite करने में use आता है|
-Normally यह command editing को बिच में ही abort करने के काम में आती है| 
-‘:q’ command फाइल को quite करने में use आता है |
-EX mode में current line number को (.) dot से जबकि अंतिम line को $(dollor)sign से represent करते है|
command फाइल में save तथा quite करने का सबसे बढ़िया तरीका ‘zz’ command का use किया जाये |


4.4 The repeat factor:-command mode में तथा इनपुट mode में use में लाये जाने वाले command को यदि एक से ज्यादा बार एक ही समय में चलाना है तो repeating factor का use करके यह वर्क किया जाता है|
-repeating factor एक numeric value होती है जिसे command के बाद execution के समय command के साथ perform किया जाता है|

FOR EXAMPLE:-

-simple ‘dd’ command single line को delete करता है परन्तु यदि यूजर को 4 lines delete करनी हो तो यूजर को ‘dd’ command को 4 time एक्सीक्यूट करने के लिए ‘4dd’ का use किया जा सकता है
– 4 respectively repeating factor है|
-इसका use अन्य command के लिए भी किया जा सकता है|


4.5 Command mode:--vi editor का यह default mode होता है|
-इसमें press की गयी हर एक key एक command की तरह ही interpret होती है|
-उन्हें हम vi editor पर print होते नहीं देख सकते है|
-इस mode में हम copy,delete etc करने का कार्य कर सकते है|

4.6 Using operators in deleting and copying tex:--vi editor में किसी text को delete करने के लिए command ‘:x’ का use करते है|
-command ‘x’ current cursor position के character को delete कर देते है |
-एक से ज्यादा character को delete करने के लिए repeating factor का भी use किया जाता है|
-उसी प्रकार command ‘dd’ current position की पूरी line को delete करने के लिए भी repeating factor का use किया जा सकता है|
*copy करने के लिए command ‘yy’ का use किया जाता है|
-‘yy’ command repeating factor के साथ भी use में लाया जा सकता है|

EXAMPLE:-

4yy
अर्थात text 4 line तक copy हो जायेगा|

COPYING TEXT:-

-‘yy’ command कम्पलीट line की copy करता है|
-‘yy’ द्वारा copy की गयी line को न्यू location पर place करने के लिए हम commad p/p का use करते है|
-‘p’ copy की गयी text को cursor के उपर तथा नीचे ‘t’ को place करवाने में use लिया जाता है |
-जिस प्रकार ‘:x’ command text को cut करता है जिससे वह text अपने स्थान से हट जाता है तब cut की गयी text line को न्यू place पर command p तथा P द्वारा put किया जाता है|

4.7 Navigation:-vi editor में navigation key(k,j,h,l) को use करके use करके cursor को movement करवाया जाता है|

-यह movement single character space जितना ही होता है|
-repeating factor को use करके यह space बदाय भी जा सकता है|

EXAMPLE:-

3h
यह cursor के three places leftकी ओर से move करवाता है|
7k
यह cursor को सेवेन lines upward करवाता है|


1.Word navigation:-word navigation के लिए key (b,e,w) का use होता है|
b-word को begining में cursor को पहुचता है|
e-word के ending में cursor पहुचता है |
w-cursor को forword begining of word.


2.Moving lines:-cursor को line के begining या end of the line में move करने के लिए zero(0),:$ तथा ^ का used होता है| 0:-cursor moves in begining of line 
$:-cursor move in end of line.


3.Moving Between line:-यदि हमें हमारे cursor की position को chek करना है तो हम “ctrl” के साथ ‘g’ press करते है |इसके बाद screen पर output निम्न प्रकार से show होगा | “sa/s/ip.pl”[modified]line 403 of 541—74%—-
sa/s/ip.p:-current फाइल name 
403:-current line no.
इसके लिए यदि हमें line no 40 पर जाना हो begining of फाइल में जाना हो या फाइल के अंतिम line पर जाना हो तो क्रमश निम्न command देने होगे|

1.40G:- goes to line number 40
2.1G:- goes to line number 1 at beginning
3.G:-goes to end of file


4.8 Pattern search:-यह command mode में pattern ko search करने के लिए use में आता है | -vi editor में search तथा replacement के लिए :$ command का use होता है|
मात्र search करने के लिए ‘/’ का या ?(question mark)का use होता है|
यहाँ निम्न महत्वपूर्ण बातो का ध्यान में रखना होगा|
*’/’:- front slash forward direction में pattern को search करता है|
*?:-question mark रिवर्स डायरेक्शन में pattern को search करता है|
command function
/ -pat Searches forward for pattern pat.
? -pat Searches backward for pattern pat.
n -Repeat search in same direction of previous search without repeat factor.
N -Repeat search in direction opposite to previous search
अर्थात search को repeat करवाने के लिए स्माल n तथा capital N का use होता है|
4.9 Joining Lines:-किसी भी document में line change करने के लिए हमें enter key press करनी होती है|
-enter key line में न्यू line character को introduce कर देती है|
एक document में अनेक lines हो सकती है|
उपरोक्त अनेक lines को एक ही बनाना हो तो इसके लिए command ‘j’ का use किया जाता है|
‘j’ command document में current line तथा next line के मध्य के ‘/n’ को remove कर देता है |

EXAMPLE:-

J- #, 4J joins following 3 lines with current one.


4.10 Undo, Repeating the last command:--last command को या last mode change को undo करने के लिए command ‘u’ का use किया जाता है यह command mode का ही command होता है|
-यह command ‘u’ single line के लिए use में आता है|
-single line में अनेक changes कर दिए गए हो थो उन सारे changes को एक साथ undo करने के लिए command ‘U’ का use किया जाता है|
*यहाँ यह बात ध्यान रखने योग्य है की command ‘U’ undo करने से पहले line को छोड़ा नहीं गया हो|
*last command को रीक्सेचुते करने के लिए dot(.) command का use करते है|
*(.)dot command दोनों mode क्रमश इनपुट तथा command mode दोनों ही चलाया जा सकता है|


4.11 Moving text from one file to another file:-mv command एक command line utility है जो फाइलों या directory को एक स्थान से दूसरे स्थान तक ले जाती है। यह single files, multiple files और directorys को move करने में support करता है। यह overwriting से पहले prompt दे सकता है और इसमें केवल उन फाइलों को transfer करने का option है जो destination की तुलना में नई हैं।mv command का उपयोग करके किसी फ़ाइल को transfer करने के लिए फ़ाइल का नाम और फिर फ़ाइल का नया नाम पास करें। निम्न उदाहरण में फ़ाइल foo.txt का नाम बदलकर bar.txt रखा गया है।
ls
foo.txt
mv foo.txt bar.txt
ls 
bar.txt
Mv command का उपयोग करके directory में फाइल को transfer करने के लिए फाइल का नाम और फिर directory पास करें। निम्नलिखित उदाहरण में फ़ाइल foo.txt को directory bar में ले जाया जाता है। tree -F .
.
├── bar/
└── foo.txt
mv foo.txt bar
tree -F 
.
└── bar/
└── foo.txt
mv command का उपयोग करके कई फाइलों को transfer करने के लिए फाइलों के नाम या destination के बाद एक pattern pass करें।
mv file1.txt file.2.txt file3.txt folder
The following example is the same as above but uses pattern matching to move all files with a .txt extension.
mv *.txt folder

-यहाँ address single या pair of number भी हो सकता है |जिन्हें (.)dot द्वारा separate करके लिखा जाता है|
$s/abc/xyz/g
