Examples:-Some simple examples using normal pattern matching:
• Pattern “Hello world” matches
o Hello world
• Pattern [Hh]”ello world” matches

o ⇒ hello world
• Pattern Hello* matches (for example)
o ⇒ Hello world
o ⇒ Helloworld
o ⇒ HelloWoRlD
o ⇒ Hello
• Pattern Hello world[[:punct:]] matches (for example)
o ⇒ Hello world!
o ⇒ Hello world.

o ⇒ Hello world?
• Pattern [[.backslash.]]Hello[[.vertical-line.]]world[[.exclamation-mark.]] matches (using collation sybols)
o ⇒ \Hello|world!

5.6 I/O Redirection:-Output redirection:-सामान्य रूप से standard output के लिए intended command से output को इसके बजाय आसानी से एक फाइल में बदल दिया जा सकता है। इस capability को output redirection के रूप में जाना जाता है। यदि notation> फ़ाइल को किसी भी कमांड में जोड़ा जाता है जो सामान्य रूप से standard output पर अपना output लिखते हैं, तो उस कमांड का आउटपुट आपके टर्मिनल के बजाय फाइल करने के लिए लिखा जाएगा। User फ़ाइल में कमांड के पूर्ण आउटपुट को redirection करने वाले कमांड की जांच करें।
$ who > users
terminal पर कोई output दिखाई नहीं देता है। ऐसा इसलिए है क्योंकि आउटपुट को डिफ़ॉल्ट standard आउटपुट डिवाइस (टर्मिनल) से निर्दिष्ट फ़ाइल में redirect किया गया है। आप संपूर्ण सामग्री के लिए उपयोगकर्ताओं की फ़ाइल देख सकते हैं:–
$ cat users
oko tty01 Sep 12 07:30
ai tty15 Sep 12 13:32
ruth tty21 Sep 12 10:10
pat tty24 Sep 12 13:07
steve tty25 Sep 12 13:03
$
यदि किसी कमांड का आउटपुट फ़ाइल में रीडायरेक्ट हो जाता है और फ़ाइल में पहले से ही कुछ डेटा होता है, तो वह डेटा खो जाएगा। निम्नलिखित उदाहरण पर विचार करें:-
$ echo line 1 > users
$ cat users
line 1
$
आप किसी मौजूदा फ़ाइल में आउटपुट को जोड़ने के लिए >> ऑपरेटर का उपयोग कर सकते हैं:–
$ echo line 2 >> users
$ cat users
line 1
line 2
$


Input Redirection:-जिस तरह एक कमांड के आउटपुट को किसी फाइल में redirect किया जा सकता है, उसी तरह एक फाइल से कमांड के इनपुट को रीडायरेक्ट किया जा सकता है। जैसा कि अधिक से अधिक character> output redirection के लिए उपयोग किया जाता है, कम-से-कम character <का उपयोग कमांड के इनपुट को redirection करने के लिए किया जाता है। सामान्य रूप से standard इनपुट से उनका इनपुट लेने वाली commands का इनपुट उनके इनपुट को इस तरह से फ़ाइल से redirection कर सकता है। उदाहरण के लिए, ऊपर उत्पन्न फ़ाइल user में लाइनों की संख्या की गणना करने के लिए, आप निम्नानुसार कमांड execute कर सकते हैं:-
$ wc -l users
2 users
$
Execution के बाद, आप निम्न आउटपुट प्राप्त करेंगे। आप फ़ाइल user से wc command के standard इनपुट को redirection करके फ़ाइल में लाइनों की संख्या की गणना कर सकते हैं:–
$ wc -l < users
2
$


5.7 Command argumentes and parameters:-unix shell का उपयोग command चलाने के लिए किया जाता है, और यह user को इन command के run time argument पास करने की अनुमति देता है। इन arguments को command lines parameters के रूप में भी जाना जाता है, जो user को कमांड के flow को control करने या command के लिए इनपुट डेटा को execute करने की अनुमति देता है।
command चलाते समय, user command line में parameters की एक variable number को pass कर सकता है।
command script के भीतर, पास किए गए parameters positional ’का उपयोग करके पहुंच योग्य हैं। ये $ 0 से $ 9 तक होती हैं, जहाँ $ 0 command के नाम को refer करता है, और $ 1 से $ 9, नौवें पैरामीटर के माध्यम से पहला होता है, जो इस बात पर निर्भर करता है कि वास्तव में कितने पैरामीटर पारित किए गए थे।


Example:-Here $0 would be assigned sh
$1 would be assigned hello
$2 would be assigned how
And so on …

5.8 command substitution:-shell Substitution का कार्य करता है जब यह एक अभिव्यक्ति का सामना करता है जिसमें एक या अधिक special character होते हैं।

Example:-यहां, variable के printing value को इसके value द्वारा substitute किया जाता है। उसी समय, “\ n” को एक नई लाइन द्वारा substitute किया जाता है –
#!/bin/sh
a=10
echo -e “Value of a is $a \n”
आपको निम्न परिणाम प्राप्त होगा। यहाँ -e विकल्प बैकस्लैश एस्केप की व्याख्या को सक्षम करता है।
Value of a is 10
Following is the result without -e option−
Value of a is 10\n
The following escape sequences which can be used in echo command −

आप backslash ESC (डिफ़ॉल्ट) की interpretation को disable करने के लिए e option का उपयोग कर सकते हैं। आप एक नई लाइन के सम्मिलन को disable करने के लिए एक एन option का उपयोग कर सकते हैं।


Command Substitution:-command substitution वह mechanism है जिसके द्वारा shell command के एक set को perform करता है और फिर command के स्थान पर उनके आउटपुट को substitute करता है।


Syntax:-The command substitution is performed when a command is given as −
`command`


Example:-command substitution आम तौर पर एक variable के लिए command के outputको assign करने के लिए उपयोग किया जाता है। निम्न में से प्रत्येक उदाहरण command substitution दर्शाता है #!/bin/sh
DATE=`date`
echo “Date is $DATE”
USERS=`who | wc -l`
echo “Logged in user are $USERS”
UP=`date ; uptime` echo “Uptime is $UP”

Upon execution, you will receive the following result−Date is Thu Jul 2 03:59:57 MST 2009
Logged in user are 1
Uptime is Thu Jul 2 03:59:57 MST 2009
03:59:57 up 20 days, 14:03, 1 user, load avg: 0.13, 0.07, 0.15


5.9 shell variable:-एक variable real data के लिए एक pointer से ज्यादा कुछ नहीं है। shell आपको variable बनाने, assign करने और हटाने में enable बनाता है।


Variable Names:-एक variable के नाम में केवल अक्षर (A से Z या a से z), संख्या (0 से 9) या underscore वर्ण (_) हो सकते हैं।
convention द्वारा unix shell variable का नाम UPPERCASE में होगा।
निम्नलिखित उदाहरण valid variable नाम हैं –
_ALI
TOKEN_A
VAR_1
VAR_2
Following are the examples of invalid variable names −
2_VAR
-VARIABLE
VAR1-VAR2
VAR_A!
जैसे आप अन्य characters का उपयोग नहीं कर सकते हैं !, *, या – यह है कि इन वर्णों का शेल के लिए एक विशेष अर्थ है। 


Defining Variables:-Variables are defined as follows−
variable_name=variable_value
For example −
NAME=”Zara Ali”
उपरोक्त variable चर NAME को define करता है और इसे “Zara Ali” मान प्रदान करता है। इस प्रकार के variable को scalar variables कहा जाता है। एक scalar variables एक बार में केवल एक मान रख सकता है।
shell आपको एक variable में किसी भी value को store करने में सक्षम बनाता है। उदाहरण के लिए –
VAR1=”Zara Ali”
VAR2=100


Accessing Values:-एक variable में store value तक पहुँचने के लिए, doller के चिन्ह ($) के साथ उसका नाम prefix करें –
उदाहरण के लिए, निम्न script define variable NAME के मूल्य तक पहुंच जाएगी और इसे STDOUT पर print करेगी –
NAME=”Zara Ali”
echo $NAME
The above script will produce the following value −
Zara Ali


Read-only Variables:-shell केवल-read के लिए केवल order का उपयोग करके variable को read के लिए mark करने का एक तरीका प्रदान करता है। variable को केवल read के लिए mark करने के बाद, इसका मान नहीं बदला जा सकता है। उदाहरण के लिए, NAME का मान बदलने का प्रयास करते समय निम्न script एक error उत्पन्न करता है – NAME=”Zara Ali”
readonly NAME
NAME=”Qadiri”
The above script will generate the following result−
/bin/sh: NAME: This variable is read only.


Unsetting Variables:-variable को हटाना या, चर को चर की list से हटाने का निर्देश देता है जो इसे track करता है। एक बार जब आप एक variable को unset कर देते हैं, तो आप variable में store मान तक नहीं पहुँच सकते।
निम्नलिखित एक define variable का उपयोग करने के लिए syntax है जो कि unset command का उपयोग करता है
unset variable_name
The above command unsets the value of a defined variable. Here is a simple example that demonstrates how the command works −
NAME=”Zara Ali”
unset NAME
echo $NAME
उपरोक्त उदाहरण कुछ भी नहीं छापता है। आप बिना read किए गए mark variable को unset command का उपयोग नहीं कर सकते।


Variable Types:-जब कोई shell चल रहा होता है, तो तीन मुख्य प्रकार के variable मौजूद होते हैं 


1.Local Variables:−एक local variable एक variable है जो शेल के present उदाहरण के भीतर मौजूद है। यह shell द्वारा शुरू किए गए programs के लिए उपलब्ध नहीं है। वे command prompt पर सेट किए गए हैं।


2.Environment Variables:-एक Environment Variables खोल के किसी भी child की process के लिए उपलब्ध है। कुछ programs को सही ढंग से काम करने के लिए Environment Variables की आवश्यकता होती है। आमतौर पर, एक shell script केवल उन्हीं Environment Variables को परिभाषित करती है जो उन programs द्वारा आवश्यक होते हैं जो इसे चलाता है।


3.Shell Variables:-एक Shell Variables एक विशेष Variables है जो Shell द्वारा सेट किया गया है और Shell द्वारा सही ढंग से कार्य करने के लिए आवश्यक है। इनमें से कुछ variable Environment Variables हैं जबकि अन्य Local Variables हैं।
