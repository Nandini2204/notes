1. sender end पर SMTP server user computer द्वारा भेजे गए message को प्राप्त करता है | 
2. sender end वाला SMTP server receiver के SMTP server को message transfer कर देता है | 
3. receiver का computer receiver के SMTP server से email message को प्राप्त कर लेता है | इस कार्य के लिए POP या push आदि protocols के काम में लेता है |

2.)server इसके बाद एक acknowledgement भेजेगा जिसमे उसका DNS name भी होगा | 
3.)client hello. command server को भेजेगा और अपनी identification भेजेगा | 
4.)अब client server को email message भेजेगा |

6.)A client recipient की list भेजेगा| 
7.)अन्य operation के बाद में connection quite हो जाता है | 

6.3 Brief Overview:-

6.3.1 Privacy Enhanced Mail(PEM):-PEM एक mail security standard है | जो IAB द्वारा adopt किया गया है | तथा यह इंटरनेट पर secure electronic mail उपलब्ध करवाता है | यह केवल ऐसे मानक है | जो इंटरनेट पर source इ-मेल transfer करते है | 
PEM तीन प्रकार के cryptography सिस्टम support है |
(1.) Encryption
(2.) Non-repeadation 
(3.) Integrity 
PEM को operation के level निम्नानुसार है | 
1. Connonical conversion 
2. Digital Signature
3. Encryption 
4. Base 64 Encoding 
1. Connonical conversion :- साधारणतया sender और reciever के कंप्यूटर architecture एंड os different होते है | PEM प्रत्येक इ -मेल message को abstract connonical representation में बदल देता है | इसमें इ – मेल uniform और independent फॉर्म में travel करता है | 
2. Digital Signature :- इसमें digital signature का process किया जाता है | 
3. Encryption :- इस step में original इ -मेल तथा digital signature एक symmetric key से encrypt किये जायेगा | 
4. Base 64 Encoding :- यह PEM का अंतिम step है | इसे radix 64 encoding भी कहते है | इसमे binary input print table characteer ,output में convert हो जाता है |


6.3.2 Pretty good privacy:-इसका पूरा नाम pretty good privacy है। यह एक encryption तथा decryption कंप्यूटर प्रोग्राम है जो कि इंटरनेट में ईमेल messages को encrypt तथा decrypt करने के लिए प्रयोग में लाया जाता है। PGP, public key cryptography की विधि पर आधारित है। इसमें दो keys का प्रयोग किया जाता है। message को encrypt करने वाली key को public के लिए रखा जाता है और message को decrypt करने वाली key को secret रखा जाता है। इसका सबसे बड़ा फायदा यह है कि हम इसका प्रयोग आसानी से कर सकते है तथा यह एकदम मुफ़्त है। PGP का अविष्कार Philip Zimmerman ने किया था।

6.3.3 Secure multipurpose secure Internet mail Extensions:-S/MIME basic email system extant को करता है | यह binary फाइल्स को basic email system के द्वारा भेजना permit करता है | जैसा की हम देखते है कि email system में from,to,date,subject, etc. email header मस्सागेकी body के बारे में सूचना देता है | ये header निम्न है –
1. MIME version :-इसमें MIME का version number लिखा होता है | 
2. content type :-MIME सात प्रकार के contant types तथा 15 contant subtypes specify करवाता है |
types : (1) Text (2)multipart (3)message (4)image (5)vedio (6)Audio (7)Application
3. content transfer encoding :- इसमें body के message को transfer करने के लिए कोनसी encoding काम में ली है | ये बताता हे दूसरे शब्दो में message को 1 और 0 में encode करने के लिए कौनसा method काम में लिया गया है | 
4. content ID :-यह MIME entities को identify करने के लिए काम में आएगा | 
5.contant description :-यह तब काम में आता है जब body redable ना हो | for example :- vedio |
