2.1 Attacks: A general and technical view:-General view:-

1.Criminal Attack:-इसमें attacker का मुख्य उद्देश्य computer system पर attack करके अधिक financial gain प्राप्त है|

2.Publicity Attack:-ऐसे attack में attacker publicity प्राप्त करना चाहता है जैसे कि news paper पर,news channel पर आदि| इसमें Criminal attack के गुण पाए जाते है|

3.Legal Attack:-इसमें attacker यह कोशिश करता है कि न्यायालय के सामने यह सिद्ध करे कि computer system की security में कमी थी| 

Technical View:-

1.Theoritical:-

(1)Interception:-इसमें unauthorized party किसी resources को access करके लाभ प्राप्त करता है| उदाहरण के लिए data copy करना,network को सुनना| 
(2)Fabricatio:-इसमें illegal object को computer system पर create किया जा सकता है| उदाहरण के लिए attackers database में fake records को add कर देगा| 
(3)Modification:-इसमें attacker database की value को modify करते है| 
(4)Interruption:-इसमें resources unavailable हो जाता है या unusable हो जाता है,या lost हो जाता है| उदाहरण के लिए program को erase कर देना,operating system के component या data को unusable बना देना| 


2. Practical View:-

(1)Applicationable attacks:-इसमें attacker applicationable label पर attack करता है अर्थात attacker particular application को process करने की,modify करने की या prevent करने की कोशिश करता है| 
उदाहरण के लिए credit card की information प्राप्त करना transaction amount को change कर देना| 
(2)Network attacks:-इस attack का उद्देश्य है कि computer network को धीमा कर दिया जाये या पूरी तरह से ठप कर दिया जाये|

2.2 Active and passive attacks:-

active attacks:-एक active attack system resources को बदलने या उनके operation को affect करने का प्रयास करता है।Active attack में data stream के कुछ modification or incorrect statement का creation involve है।

Types of active attack:-
1.Masquerade:- masquerade attack तब होता है जब एक इकाई (unit)अलग इकाई होने का दिखावा(pretend) करती है। एक excuse attack में active attack के अन्य रूप में से एक शामिल है।

2.Modification of messages:- इसका मतलब है कि किसी message का कुछ part changeकिया गया है या उस message को unauthorized effect उत्पन्न करने के लिए delayed या rearranged किया गया है। उदाहरण के लिए, एक message जिसका अर्थ है “confidential file X को read की अनुमति दें” has been modified as “Allow confidential file X to be read”है।

3.repudiation:- यह हमला sender या receiver द्वारा किया जाता है। sender या receiver बाद में denies कर सकता है कि उसने एक message भेजा है या प्राप्त किया है। उदाहरण के लिए, Customer अपने bank से “किसी को amount transfer करने के लिए” कहता है और बाद में sender (ग्राहक) से denies करता है कि उसने ऐसा rquest किया है। यह revenge है। 


4.Reply:- This includes passive capture of a message and subsequent broadcast to create an authorized effect

5.Denial of service:- यह communication facilities के सामान्य उपयोग को रोकता है। इस attack में एक specific target हो सकता है। उदाहरण के लिए, एक entity किसी particular destination के लिए निर्देशित(suppress) सभी message को दबा सकती है।

service deprivation का एक another form नेटवर्क को disable करने या message को overload करने के लिए एक entire network के disruption का कारण है ताकि performance degraded दिखाया जा सके।

Passive attacks:-एक passive attack system से in+formation का use or सीखने का प्रयास करता है लेकिन system resources को effect नहीं करता है। Passive attacks के transmission की monitoring or monitoring करने की प्रकृति(nature) में हैं। opponent’s का goal information प्राप्त करना है जिसे transmitted किया जा रहा है।

Types of passive attacks:-


1.The release of message contect :- Telephone conversation, एक electronic mail message या transmitted file में sensitive or confidential information हो सकती है। हम एक opponent को इन prevent की content को सीखने से रोकना चाहते हैं। 

2.Traffic analysis:- मान लीजिए कि हमारे पास information का एक method (एन्क्रिप्शन) है, ताकि attacker ने भले ही message पर capture कर लिया हो, लेकिन message से कोई भी information नहीं निकाल सकता है। Opponent determine the position और identity का निर्धारण कर सकता है और message की frequency और length का आदान-प्रदान(exchange) कर सकता है। यह information nature of communication का अनुमान लगाने में उपयोगी हो सकती है।

2.3 Programs that attacks :-

2.3.1 Virus:-virus computer program होते है जो अपने आप को दूसरे legitimate program के साथ attach कर लेते है और इन virus के कारण computer system या network को damage करने का कारण बनती है| virus अपने जीवनकाल में निम्न 4 phases से गुजरता है| 
1. Dormat Phase:-इसमें virus निष्क्रिय अवस्था में रहता है| यह virus किसी निश्चित action या event के आधार पर activate हो जाता है| यह एक optional phase है| 
2. Propagation Phase:-इस phase में virus अपने आप की copy बनाता है और प्रत्येक copy अपने आप की copy बनती है और इस प्रकार virus फैलता है|
3. Triggering Phase:-Dormat virus इस phase में तब चलाया जाता है जब Dormat virus को जिस शुरुआत का इंतजार था वह पूरा हो गया है अर्थात कोई action या events का वह इंतजार कर रहा हो वह उपलब्ध हो गया हो|
4. Execution Phase:-यह virus का actual work है| जो harmless हो सकता है|
virus को निम्नलिखित category में विभाजित कर सकते है-
(1)Parasitic Virus:-यह virus की most common form है इसमें virus executable files के साथ अपने आप को attach कर लेता है और अपनी copies को बढ़ाता है| जब यह infacted file execute हो जाती है तो यह virus फिर दूसरी executable file के साथ attach हो जाता है और इस प्रकार फैलता जाता है| 
(2)Memory Resident Virus:-इस प्रकार का virus सबसे पहले अपने आप को main memory area से attach हो जाता है और phase प्रत्येक execute होने वाले program को infact करता है| 
(3)Boot Sector Virus:-इस type का virus disk एक master boot records को infact करता है तथा जब operating system computer को boot करवाता है उस वक्त यह spread(फ़ैल) जाता है| 
(4)Stealth Virus:-यह virus intelligent होता है जो anti-virus s/w program से अपने आप को detect होने से बचाता है 
(5)Polymorphic Virus:-यह virus अपने signature अर्थात identity को प्रत्येक execution के साथ बदलता रहता हैइस प्रकार इसे detect करना आसान नहीं है| 
(6)Netamorphic Virus:-यह virus polymorphism virus की तरह ही अपने signature को बदलता ही है परन्तु इसके साथ हर बार अपने आप key re-right करता है जिससे इस virus को detect करना बहुत मुश्किल हो जाता है| 
(7)Macro Virus:-यह virus कोई विशेष application s/w जैसे कि microsoft word ,microsoft excel आदि को effect करता है| 


2.3.2 Worm :-

worm कोई destrective action नहीं करता है| यह केवल computer system के resource को consume कर देता है जिससे वो system down हो जाता है| यह किसी program के साथ attach हो जायेगा और बार-बार अपने आप को replicate करेगा और यह replication इतना अधिक हो जायेगा कि computer system और network बहुत ज्यादा slow हो जायेगा और finally halt हो जायेगा| 

Trojan horse:-Trojan horse attacker को computer system या network की कुछ confidentional information पहुंचाने का काम करता है| Trojan horse login screen के code में बैठा रहेगा| जब user user id या password enter करेगा तो यह Trojan horse इन details capture कर सकता है और इस information को attacker को भेज देता है परन्तु इस बारे में user को कुछ पता नहीं चल पायेगा|

Java applets form sonmicrosystem के द्वारा तथा activex control from microsoft यह small client side program होते है जो कभी-कभी security problems उत्पन्न कर देते है| 


2.3.4 Cookies:-cookies एक या एक से अधिक इनफार्मेशन के भाग होते है जो client computer की disk पर text file में text stream के रूप में store रहते है| web server web browser को एक cookies भेजता है तथा web browser client computer की hard disk पर store कर लेता है| इसके बाद जब भी server को http request भेजनी होती है तो browser cookies की एक copy server को भेजता है| यह identification purpose के लिए काम में आती है| 
Scripts:-web page html से बनाया जाता है|इस language मे tag होते है|tag <>से start होते हो और से end होते है| इस text की boundaries के बीच जो display करानी है|वो लिखी जाती है|जैसे ये text format को bold करने के काम आता है| एक web page में html tags के अलावा client side scripts भी होती है|ये छोटे program होते है जो scripting language जैसे java script VB.scripts और J script आदि मे लिखे होते है|जो web browser द्वारा client computer में execute होते है|
Example:- कोई user online book shop को visit करता है|मान लीजिये ये website एक condition ये डाल दे की कोई user कम से कम 3 books से कम order नहीं कर सकता| ऐसी अवस्था में एक web page small java script के माध्यम से यह condition को पूरी होने के लिए execute करवाता है और यदि condition पूरी नहीं होती है तो वह user को आगे proceed होने से रोक देगा| 
कई बार यह script danger होती है क्योकि यह छोटे-छोटे program होते है जो client computer पर कई सारे action perform कर है|

(iv)fourth generation :-यानि की scanner activity monitor इन सभी पर आधारित actions लेते है|


2.5 Specific attacks:-

2.5.1 Sniffing and Spoofing:-packet sniffing on going convertation पर passive attack है। इसमें attacker को convertation hi-jack करने की आवश्यकता नहीं है इसमें attacker पास हो रहे packets को simply observe करता है। ऐसे attack को दो तरीके से रोका जा सकता है। 
1. travelling data को किसी तरह से encode कर दो। 
2. transmittion ink को ही encode कर दो।
Packet Spoofing:-इस technique में attacker faise source address से packet send करता है। जब ऐसा होता है तो reciever उस forget address पर reply करता है। ऐसा होने पर तीन संभावनाए बनती है।
(1) attacker reply को intersact करे तथा information को use करके attack करे। 
(2) attacker reply देखा न पाए। यदि attacker का उद्देश्य reply को देखना नहीं है बल्कि उसका intention denile of service है। 
(3) attacker को reply नहीं चाहिए। इसमें attacker का उद्देश्य reciever को परेशान करना या उसको confuse करना है।


2.5.2 Phishing:-

(1) इसमें attacker एक fake website बनता है जो real website जैसी दिखाई देती है 
(2) अब attacker बैंक के customber पर कई प्रकार से attack कर सकता है। जैसे -attacker बैंक के किसी customber को एक e-mail send करे और यह e-mail ऐसा लगे की bank से आया हो। अब यह fake e-mail बैंक को यह चेतावनी दे की बैंक के computer system पर attack हो गया है और हम computer को नया password issue करेंगे और customber के pin को verify भी करेंगे। अतः इस कार्य को करने के लिए इस e-mail में दिए गए URL पर visit करे|
(3) अब customber attacker के instation के अनुप्रान्त attacker की website का url खोलेगा और attacker की website पर चला जायेगा। अब उस fake website पर customber को अपने id और password आदि enter करने को कहा जाएगा। क्योकि यह fake website original वेबसइट जैसी लग रही है। इसलिए customber यह गोपनीय सूचनाएं enter कर देगा। attacker इन गोपनीय सूचनाओं का उपयोग कर attack करके transition आदि कर सकेगा।


2.5.3 Pharming or DNS Spoofing:-

1. मान लीजिए एक merchant है उसकी site का domain name http://www.bob.com है और उसका IP address 100.10.10.20 है| अब DNS server में bob की DNS entry निम्न अनुसार होगी है|
http://www.bob.com | 100.10.10.20
2. मान लीजिए एक attacker इस bob के IP address को hijack करता है और इस IP address की जगह अपना IP address लिख देता है| इस प्रकार मान लीजिये attacker Alice के ISP द्वारा maintain किये गए DNS server में bob की entry इस प्रकार करता है| 
3. जब Alice bob की website से communicate करना चाहेगा तो DNS server उसे attacker की site पर ले जायेगा|
4. इस प्रकार Alice attacker से communicate करना शुरू कर देगा| 
इस प्रकार के attack को रोकने के लिए DNS Sec protocol काम में लिया जाता है|
