1.1 Introduction:-पहले हमे security से related various concept and technical issues को समझ लेना चाहिए| means यह समझने से पहले की हमे कैसे protected (secure) करना है यह समझना चाहिए की हमे किसको protect करना है| जब हम computer, computer networks and the biggest network का use करते है तो बहुत सारे dangers होते है| security में सबसे पहला question यह आता है कि why is security required?

1.2 Need of security:-Network को attackers और hackers के against सुरक्षा की आवश्यकता है। network security में two basic security शामिल हैं। first data information की security है यानी information को unauthorized access और loss से protect करने के लिए और दूसरा है computer security यानि data की सुरक्षा और hackers को नाकाम(thwart) करना। यहां network security का मतलब केवल किसी network या network के किसी एक network मे ं सुरक्षा से नहीं है। अब हमारी network security की जरूरत दो जरूरतों में टूट गई है। एक information security की आवश्यकता है और अन्य computer security की आवश्यकता है। internet या किसी organization के किसी भी network पर, हजारों important information का प्रतिदिन exchanged किया जाता है।Attackers द्वारा इस information का misused किया जा सकता है। निम्नलिखित कारणों से information security की आवश्यकता है। net पर ही secret information user की security के लिए। किसी अन्य person को इसे देखना या access नहीं करना चाहिए। Unauthorized users द्वारा accidently or intentionally unwanted editing, से जानकारी को protect करने के लिए। information को loss से बचाने के लिए और इसे अपने destination तक सही तरीके से पहुंचाने के लिए।
specific situation में sender द्वारा denial से protect के लिए किसी भी node द्वारा प्राप्त message की acknowledgement के लिए manage करने के लिए। उदाहरण के लिए, एक customer को व्यापक रूप से कुछ shares XYZ खरीदने का आदेश दें और दो दिनों के बाद order के लिए मना कर दें क्योंकि rates नीचे जाती हैं। एक users को third message के नाम के साथ किसी अन्य user को कुछ message भेजने के लिए restrict करना। उदाहरण के लिए,users X अपनी स्वयं की interest के लिए एक message बनाता है जिसमें कुछ favourable instructions होते हैं और इसे user Y को इस तरह sends करता है कि Y organization के manager Z से आने वाले message को स्वीकार करता है।
urgency के case में message को transmittion lines / route में unwanted delay से बचाने के लिए इसे आवश्यक destination तक पहुंचाने के लिए। network में data packet या information packet को भटकने से बचाने के लिए infinitely रूप से long time के लिए और इस प्रकार line में increaing congestion के कारण destination machine कुछ internal faults के कारण इसे पकड़ने में fail रहती है।
network security के एक अन्य भाग में computer security include है। computer security का मतलब network के कारण होने वाली unwanted damage से आपके कंप्यूटर सिस्टम की रक्षा करना है। इस तरह के नुकसान का एक प्रमुख कारण virus और spywares हैं जो आपकी hard disk से सारी जानकारी मिटा सकते हैं या कभी-कभी वे enough destructive भी हो सकते हैं और hardware की समस्या भी पैदा कर सकते हैं। निश्चित रूप से network को इस प्रकार के हानिकारक software से protect किया जाना चाहिए । जो लोग जानबूझकर इस तरह के सॉफ्टवेयर को नेटवर्क पर डालते हैं उन्हें hackers कहा जाता है। चूंकि नेटवर्क कंप्यूटर इसका हिस्सा है, इसलिए hackers से कंप्यूटर सुरक्षा भी नेटवर्क सुरक्षा का एक हिस्सा है। hackers से कंप्यूटर सुरक्षा की जरूरतें इस प्रकार हैं: –
* infected files से virus की replicating और capturing से इसे protect किया जाना चाहिए।
* इसे woms और bomb से उचित सुरक्षा की जरूरत है।
*trpjan horse से सुरक्षा की आवश्यकता है क्योंकि वे आपके कंप्यूटर के लिए पर्याप्त खतरनाक हैं। 
1.3 Security Approaches →

1. Trusted System:-

वह computer security जिसमे specified security policy हो trusted system होता है| Trusted system में reference monitor term का use किया जाता है| यह एक entity है जो कि computer का logical heart होती है| यह mainly access controls से related सभी decisions लेने में responsible है| 
Naturally, reference monitor के निम्न expectations है| 
(a) It should be tamperproof.
(b) It should always be envoked.
(c) It should be small enough so that it can be independentally tested.
एक orange book(also called as the trusted system computer system Evaluation Criteria है,जो कि उस environment की national security Agency(NSA) को set of evaluation classes में define करती है| यह उन features and assurance को describe करती है जो कि trusted system से user expect करता है| assurances की highest level trusted computing basic or TCB की reduction of size(sort integrated data) provide करती है| इस context में TCB system’s security के लिए hardware, software and firmware का combination define करती है| Minimum TCB higher assurance provide करते है|

ऐसा organization जो several approaches को लेकर उसके security model कर सकता है और उसके approaches summarize कर सकता है| 

No Security:-यह एक simplest case है, वह approach जो decision लेने के लिए सभी में security implement नहीं करता है means ऐसा computer system जिसमे security न हो| 
Security through obsecurity:-इस model में, ऐसे system आते है जिनको simply secure किया जाता है क्योकि कोई भी इसके existence and contents के बारे में नहीं जानता है यह approach लम्बे time के work नहीं कर सकता है क्योकि एक attacker के पास इसके बारे में जानने के लिए बहुत से ways होते है| 
Host Security6:-यह एक अच्छी approach है| लेकिन यह trouble को अच्छी तरह scale नहीं कर सकता है| 
Network Security:-host security इसके organization को grow करने के योग्य बनाती है और इसको more diverse बनाती है| इस technique में various hosts and their services के लिए network access control करने के लिए focus किया गया है| means individual host security को प्राथमिकता दी गयी है|यह बहुत ही efficient and scalable model है | 

3. Security Management Practices:-

किसी भी place में जहाँ पर security policy होती है वहाँ पर हमेशा good security management practice की बात होती है| यदि हम कहीं भी security policy किसी भी place में बिना सोचे समझे apply करते है तो वह ज्यादा time तक secure नहीं रहती हे | यदि good security policy हो and उसका implement properly हो तो यह security management practice का एक लम्बा रास्ता होता है| means यदि एक good security को properly implement किया जाये तो वह लम्बे time तक system को secure रखती है| 
एक good security policy की generally 4 key होती है-
*Affordability:-Cost and efforts in security implementation.
*Functionality :-Mechanism of providing security.
*Cultural issues :-Whether the policy gets well with people expectations,working and beliefs.
*Legality:-Whether the policy meets the legal requirements. (a) Explanation of the policy to all concerned.
(b) Outline everybody’s responsebilities.
(c) Use simple language in all comunications.
(d) Establishment of accountability.
(e) Provision for exceptions and periodic reviews.

1.4 Principal pf Security:-हम कुछ ऐसे attacks के solution के लिए जो real life से occure होते है security के principle के बारे में जानेंगे| security के द्वारा हम attacks को अच्छी तरह समझ सकते है और यह better and possible solution लेने के लिए help provide करेगा| Security के mainly 4 principle हे जजों particular message से related है| इसके आलावा 2 principles ओर भी है but यह particular message से related नहीं है(“Access control ” and “Availability”)परन्तु सारे system से linked रहते है| 

1. Confidentiality →

Confidentiality principle के according sender द्वारा send किये गए message को वही receive करती है जिसको contents of message को access करने की permission हो | 
Example :- यदि user A another user B के लिए message send करता है and without A और B की permissions और knowledge कोई दूसरा user C उस message को access कर लेता है तो इस type के ाटकको interception कहते है| इस तरह से confidentiality loss हो जाती है|

2. Authentication →

Authentic Process यह सुनिश्चित करता है की document & message origin सही प्रकार से identify हो गया है|
Example:- मान लीजिये User C User B को internet के माध्यम से एक message भेजा| परन्तु User C अपने आपको A बताकर ये message भेजता है तो User B को कैसे पता लगेगा की वास्तव में ये message किसने भेजा|

3. Integrity →

जब Sender द्वारा भेजे गए message में कुछ परिवर्तन होकर message receptient को को प्राप्त होवे तो हम कहेंगे की message की integrity lost हो गयी है| इसलिए integrity के सिद्धांत में सुनिश्चित किया जाता है की sender द्वारा भेजा गया message हूबहू receptient को प्राप्त हो|

Example:- माना user A ने user B को कोई message send किया another user C ने without permission उस message को access करके और उसके content को change करके user B को send कर दिया है| user B को यह जानकारी नहीं होती है कि user A के द्वारा message send करने के बाद message के content change हो चुके है| इस type के attack को modification attack कहते है|

4. Non-repediation →

कई बार ऐसी situation create हो जाती है की Message भेजने वाला Message भेजने के बाद यह कह दे की मैंने message भेजा ही नहीं या मुकर जाए इसे हम repndiation कहते है|
Example:-माना user A ने bank B को internet के द्वारा funds के लिए request send करता है जबकि bank B message के according funds transfer कर देता है तो user A कहता है कि उसने funds के लिए कोई request की ही नहीं थी| Non – rependiation का Principal ऐसी संभावना को खत्म कर देता है और Sender को यह कहने की मैंने message नहीं भेजा की अनुमति नहीं देता|

5. Access Control →

Access Control का सिद्धांत यह बताता है की कौन User क्या Access कर सकता है और किस type के message को access कर सकता है| हम यह specify करते है कि user A only database के records को देख सकता है लेकिन उन्हें update नहीं कर सकता है| user B update कर सकता है| access control mechanism इस type के set up को ensure करता है|

6. Availability →

Availability का सिद्धांत कहता है की Authorised party को हर समय resources उपलब्ध होने चाहिये|
Availability एक ऐसी state है जो information को all time available करवाती है| 

Example:-एक unauthorized user C intentional action लेता है और एक authorized user A भी यही चाहता है लेकिन वे किसी server computer B से connect होने के लिए सक्षम नहीं है| यह availability के rule को तोड़ता है इस type के attack को interruption कहते है|
