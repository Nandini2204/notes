2.1Basics of applet:-Applet एक java program है जिसे web page में embed किया जा सकता है। यह web browser के अंदर चलता है और client की तरफ काम करता है। Applte को HTML page में APPLET या OBJECT tag का उपयोग करके और web serverपर host किया गया है। web siteको अधिक गतिशील और मनोरंजक बनाने के लिए सेब का उपयोग किया जाता है।

2.2 Applet life cycle:-Applet क्लास 5 important methods provide करती है, जिन्हे आप override करते है। इन methods को life cycle methods भी कहते है। एक applet program अपनी life cycle में इन सभी methods से होकर गुजरता है। ये सभी methods आपको applet program पर पूरा पूरा control provide करते है।

1.init():–

इस method में आप variables को initialize कर सकते है या फिर कोई object क्रिएट कर सकते है। किसी भी तरह का initialization आप इस method में कर सकते है। जैसे की applet का background color set कर सकते है।

2.start():–

ये मेथड init() method के बाद automatically call होता है। ये method browser में applet को call करता है।

3.stop():–

ये मेथड applet को stop करने के लिए यूज़ किया जाता है। आप जब applet वाले HTML page को छोड़ कर किसी दूसरी window पर जाते है तो ये method call होता है और applet pause हो जाता है। जब आप वापस उसी window पर आते है, तो applet फिर से शुरू हो जाता है।

4.destroy():–

जब आप browser को close करते है तो ये मेथड call होता है और applet destroy हो जाता है।

5.paint():–

इस method को आप applet में कुछ draw करने के लिए यूज़ किया जाता है। ये method start() मेथड के बाद call होता है।

2.3 Applet tag:-

Applet को किसी web page में embed करने के लिए HTML के applet tag की आवश्यकता होती है | यह applet tag applet को start करने का कार्य करता है| Applet tag के अनेक attributes होते है, जिनमे से code ,height,width tag होते है | 
< APPLET
[CODEBASE = codebaseURL]
CODE = appletFile 
[ALT = alternateText]
[NAME = appletInstanceName]
WIDTH = pixels
HEIGHT = pixels 
[ALIGN = alignment] 
[VSPACE = pixels]
[HSPACE = pixels]
>
[< PARAM NAME = appletParameter1 VALUE = value >]
[< PARAM NAME = appletParameter2 VALUE = value >]
. . .
[alternateHTML]

CODEBASE:-

यह एक optional attribute है, जो applet code का base URL को specify करने के उपयोग में आता है | यदि कोई applet remote computer में उपस्थित है, तो उसके URL को यहां लिखना होता है| 

CODE:-

यह attribute applet के compile byte code के नाम द्वारा initialize किया जाता है| byte code का extension .class होता है |

ALT:-

इस attribute में उस text message को लिखा जाता है जिसे तब display कराना हो जब applet किसी कारण से run न हो पाए |

NAME:-

एक page में अनेक applets उपस्थित हो सकते है इन्हे uniquely identify करने के लिए उन्हें कुछ नाम दिया जा सकता है| यह नाम इस attribute द्वारा दिया जाता है |

HEIGHT&WIDTH :-

यह दोनों attributes आवश्यक attribute है, जिनमे applet के display area को pixels में define किया जाता है |

ALIGN:-

यह एक optional attribute है जिसके द्वारा applet के alignment को set किया जाता है | यह alignment LEFT,RIGHT,TOP,BOTTOM,MIDDLE,BASE LINE आदि हो सकते है |

VSPACE &HSPACE :-

इसके द्वारा vertical तथा horizental spcaing दी जा सकती है, जो pixels में होती है |

Update():-

कुछ situation में हमे एक अन्य method को override करने की आवश्यकता होती है| जिसे awt द्वारा ही define किया गया होता है, यह method update method होता है| यह method तब call होता है जब कोई applet की request पर portion of window को redraw की आवश्यकता होती है| update method का default version सबसे पहले applet को background color से fill करता है, उसके बाद paint method को call करता है| यदि paint method में कोई दूसरा background color define किया गया है, तो इस process में background color change होने पर user को एक flash experience होता है| इस problem के solution में update method को applet में override किया जाता है जिससे user paint method को ही call करके इस problem को solve out कर सकते है | 
import java.awt.*; 
import java.applet.Applet; 
import java.awt.event.*; 
/*
< applet code=”UpdateExample.class” width=”350″ height=”150″> < /applet>
*/
public class UpdateExample extends Applet implements MouseListener 
{ 
private int mouseX, mouseY;
private boolean mouseclicked = false;
public void init()
{ 
setBackground(Color.black); 
addMouseListener(this); 
} 
public void mouseClicked(MouseEvent e) 
{ 
mouseX=e.getX(); 
mouseY=e.getY(); 
mouseclicked = true;
repaint();
} 
public void mouseEntered(MouseEvent e){}; 
public void mousePressed(MouseEvent e){}; 
public void mouseReleased(MouseEvent e){}; 
public void mouseExited(MouseEvent e){};
public void update(Graphics g)
{ 
paint(g);
} 
public void paint( Graphics g) 
{ 
String str; 
g.setColor(Color.white);
if (mouseclicked)
{ 
str = “X=”+ mouseX + “,” + “Y=” + mouseY;
g.drawString(str,mouseX,mouseY);
mouseclicked = false;
} 
}
} 

Repaint():-

generally applet अपनी window पर तभी write करता है जब awt के update या paint method call होते है, परन्तु यदि खुद अपने आप से information होने के कारण window को update करना पड़े| 

Example :- यदि किसी applet में कोई clock display हो रही है,तो उस clock के हर second के information change पर applet को windows को भी change करना पड़े| ऐसा paint method का उपयोग करके नहीं किया जाता है | इस प्रकार के updation के लिए method repaint() का उपयोग किया जाता है | repaint भी awt द्वारा define किया गया method है, जिसके कारण awt का run time system applet के update method को call करता है| update अपने default information में paint method को call करता है| repaint() method निम्न 4 forms में पाया जाता है-
void repaint()
void repaint (int left,int top,int width,int height) first version पूरी window को repaint करने के उपयोग में लाया जाता है| second version का उपयोग करके window के किसी portion को repaint कराया जा सकता है| repaint होने वाले portion के co-ordination repaint का second version as an argument लेता है|repaint का उपयोग applet के लिए direction होता है कि applet को जल्द कुछ repaint करना है,परन्तु system busy होने के कारण कई बार repaint द्वारा update immidiate call नहीं हो पाते है,जिसके कारण output एक flow में नहीं मिल पाता है| इस problem को solve करने के लिए repaint के 2 अन्य versions का उपयोग किया जा सकता है, जिसमे इसे time bound कर दिया जाता है| 
void repaint(longmaxDelay)
void repaint(longmaxDelay,int x,int y,int width x,int height y)
यहां max Delay mili second में टिमको represent करता है|

किसी भी window में status bar का उपयोग additional information को डिस्प्ले करने के लिए किया जाता है| यह bar applet window सबसे bottom side पर लगी हुई होती है| यदि applet status bar को message के साथ print कराना है तो उसे निम्न function का उपयोग करके कराया जा सकता है|
ShowStatus(String msg)
यहाँ msg वह String है,जिसे status bar पर print कराना है|
< applet code=”ShowStatusDemo.class” width=”300″ height=”200″>
< /applet>
import java.applet.Applet;
import java.awt.Graphics;
public class ShowStatusDemo extends Applet
{

{
g.drawString(“Hello World”, 50, 75); // using drawString() to display in applet window

}
