1.1 Intoduction:-

1.2 Features of Java:-Java के बहुत सारे features है। एक अच्छा java programmer बनने के लिए आपको ये सभी features पता होने चाहिए। आइये java के कुछ popular features के बारे में जानने का प्रयास करते है।

Compiled and Interpreted:-

ज्यादातर programming languages या तो compiled होती है या interpreted होती है। लेकिन java इन दोनों approaches को combine करती है और एक two stage system बनाती है। पहले तो java आपके program को compiler करके byte code generate करती है।
Byte code machine instructions नहीं होते है। इसलिए java second stage में bytecode को interpret करके machine code generate करती है, जिसे directly execute किया जा सकता है।

Platform Independent:-

Java को bytecode platform independent बनाता है। जब आप किसी java program को compile करते है तो वो byte code में convert हो जाता है। ये byte code किसी भी मशीन या operating system पर नहीं चल सकता है। ये सिर्फ JVM (Java Virtual Machine) पर चलता है। Java के program को run करने के लिए आपको अपने operating system पर JVM install करना पड़ता है। हर operating system के लिए अलग JVM होता है लेकिन काम सभी एक ही करते है।और वो काम होता है उस operating system के लिए bytecode को machine code में convert करना। इसलिए किसी एक operating system के JVM द्वारा generate किया गया bytecode कोई भी दूसरी JVM पर run किया जा सकता है।और फिर JVM उस operating system के लिए machine code generate कर देती है। इसलिए अलग अलग machines के लिए अलग अलग machine code generate होता है लेकिन वो एक ही byte code से generate होता है क्योंकि आपका program पहले byte code में convert होता है।

Object Oriented:-

Java एक true object oriented programming language है। Java में almost सब कुछ object है। Java में सारी information objects के रूप में store होती है।

Robust and Secure:-

यदि user experience की बात की जाए तो java एक robust language है। Java में बनाया हुआ कोई भी program अलग अलग environments में अलग अलग technologies के साथ बिना crash हुए काम कर सकता है। इसके programs कभी crash नहीं होते है। ये बहुत ही reliable language है। Java में security JVM के द्वारा provide की जाती है। Machine code generate करने से पहले JVM पर कुछ tests रन करके invalid combinations को detect करती है।

Distributed:-

Java से आप distributed applications क्रिएट कर सकते है। Distributed applications वो applications होती है जो अलग अलग networks पर होती है। और एक साथ मिलकर task perform करती है। Java में RMI के through आप दूसरे network पर available applications से interact कर सकते है।

Simple and Familiar:-

Java एक simple language है। Java के बहुत से features c और c++ से लिए गए है। Java का syntax c और c++ के जैसा है। जैसे की variable declarations, control statements और method declarations आदि। ये सब java को easy तो understand बनाते है। क्योंकि java c और c++ को ध्यान में रख कर बनायीं गयी है इसलिए ज्यादातर programmers को familiar लगती है।

Multi-threaded and Interactive:-

By default java एक multithreaded language है। कोई भी java program एक साथ कई tasks complete कर सकता है। ये feature java को fast और interactive बनाता है।

High Performance:-

यदि performance की बात की जाए तो java की performance बहुत impressive है। Java की .speed का main reason byte code है। Java का architecture इस तरह से design किया गया है की java में runtime पर overhead बहुत कम होता है।

Dynamic and Extensible:-

Java एक dynamic language है। Java run time के दौरान libraries, methods और classes से dynamic linking करने में सक्षम है। इसलिए java को नयी technologies के साथ आसानी से यूज़ किया जा सकता है।

Ease of Development:-

Java में programs develop करना बहुत easy है। Java आपको built इन libraries provide करती है जिसमे आपके यूज़ के लिए important classes होती है। इससे programmer का overhead कम हो जाता है। Programmer इन libraries को access करके आसानी से software develop कर सकता है।

Servlets भी applet की तरह small program होता है| जो web connection के server side execute होता है | Servlets किसी भी प्रकार की request को responde करने के लिए बनाया जाता है इन्हे इसलिए use किया जाता हे ताकि ये web browser की functionality को extends कर सके | इसी कारण इनका execution web server पर होता है | यह java base server side technology है | Servlets java class ही होता है जिसमे ऐसे protocol provide होते है जो किसी response को accept कर सके servlets clients server protocol का use communication के लिए करते है,परन्तु इन्हे अधिकतर HTP protocol use करके बनाया जाता है, इसी कारण servlets को कभी-कभी HTP servlets भी कहते है कोई भी s/w developers servlets use web browser पर dynamic contents को add करने के लिए करता है |किसी भी servlets को run कराने के लिए web connector use जाता है web container को servlet container भी कहते है जो एक web server esention का होता है तथा servlets से interact web container करता है| web server ही container की life cycle को manage करता है | किसी भी servlets को हम object के रूप में सोच सकते है जो request के आधार पर responde का generate करता है request reserving का कार्य भी इसी द्वारा किया जाता है|

java technology के heart में java virtual m/c lay करती है| यह एक आभाषी computer होता है,जिस पर रखे java program run करते है|JVM API तथा java classes आपस में मिलकर java program को run करने के लिए कार्य करते है|java architecture में निम्न चार interrelated technology cordinated रहती है| 
→ Java Program language 
→Java Class File Formate 
→Java App Programing Interface (API) 
→Java Virtual Machine 
जब भी हम कोई java program को run करते है तो इन चार technology का use करते है| java programming language में source file को लिखा है| compiler इस source file को class file में converts करता है| यह class file java virtual m/c पर बनती है| जब भी हम कोई program को लिखते है तो साथ ही system resources को access करते है| यह accessibility method को call करके की जाती है| यह methods किसी class में रहते है| जिन्हे जावा API या interface द्वारा implement किया जाता है| 
java architecture network oriented होने के कारण platform independent secure&network mobile होना आवश्यक है| यह सारे features उसे virtual machine के concept में आते है| Java virtual machine का main job को byte code को execute करने के अलावा class files को load करने का होता है| इसके लिए java virtual machine में class loader होता है| class loader classes को ना सिर्फ program से परन्तु java API से भी load करता है| यह loading बड़ी specific होती है क्योकि वही classes load होती है,जिसकी आवश्यकता होती हो| 
जहाँ class loader classes को load करता है वही execution engine byte code के interpreter का कार्य करता है| Simple kind का execution engine byte code को one at a time interpret करता है| परन्तु अन्य execution engine भी होते है जो ज्यादा fast कार्य करते है परन्तु उनमे memory का consumption ज्यादा होता है| 

1.8 JDK TOOLS:-

JDK में अनेक tools तथा utility एक package के form में provided रहती है जिनका उपयोग कर stand alone system या फिर client server model के लिए application को prepare किया जा सकता है इस tool kit का उपयोग करना सिर्फ application को run develop किया जा सकता है परन्तु debugging तथा testing जैसे task भी किये जा सकते है,इसमें provide tools निम्न होते है-

Appletviewer:-

इस application का उपयोग java applets को run कराने के लिए किया जा सकता है|

Javac:-

javac compiler source code को byte code में translate करने के काम में आता है| 

Java:-

यह java interpreter compiler byte code को run कराने के लिए उपयोग में लाया जाता है| 

Javadoc:-

java source code file से HTML format में documentation के काम में आता है| 

Javah:-

जैसा हम जानते है Java में Native Method का उपयोग कर अन्य language जैसे-C,C++ header files को भी access किया जा सकता है,यह native method javah tool का होता है| 

Javap:-

इसे java dis assembler कहा जाता है,जिसकी मदद से converted byte code को program description में बदला जा सकता है|

jdb:-

(java debugger) इसकी मदद से java programs में errors को debug किया जा सकता है|
