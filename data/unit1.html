1.1 Introduction to data representation:-

Data Structure इस तरह से data collecting और organising करने का एक तरीका है जिससे हम इन data पर effect तरीके से operation perform कर सकते हैं। data structure better organization और storage के लिए some relationship के rendering data structure in terms को प्रस्तुत करने के बारे में है। उदाहरण के लिए, हमारे पास कुछ data है, जिसमें player का नाम “virat” और age 26 है। यहां “virat” string data types का है और 26 integer data types का है।
हम इस data को plyer record जैसे record के रूप में व्यवस्थित कर सकते हैं, जिसमें player का name और age दोनों होंगे। अब हम file या database में player के record को data structure के रूप में collect और store कर सकते हैं। उदाहरण के लिए: “Dhoni” 30, “Gambhir” 31, “Sehwag” 33
यदि आप object oriented programming concept के बारे में जानते हैं, तो एक class भी एक ही काम करता है, यह एक single entity के तहत विभिन्न प्रकार के data collect करता है। एकमात्र अंतर, data structures को manipulate data efficiently तक पहुंचने और access करने की technique प्रदान करता है।
सरल भाषा में, data structure order किए गए data को store करने के लिए structures programmed हैं, ताकि इस पर various operation आसानी से किए जा सकें। यह memory में व्यवस्थित होने वाले data के knowledge का represent करता है। इसे इस तरह से design और implement किया जाना चाहिए कि यह complexity को कम करे और efficiency को बढ़ाए।


1.2 Review of data structures-Array,Pointer,Structure,Lists,Trees,Graphs:-

1. Array:-

एक array एक single variable में similar data types के value का एक collection है। एक array C में एक derived data type है, जो C language के fundamental data type से निर्मित है।

Pointer:-

जैसे हमारे पास array of integers, array of pointers आदि हैं,हमारे पास structure variables की array भी हो सकती है। और structure variable के array को efficiently से उपयोग करने के लिए, हम structure types के pointers का उपयोग करते हैं। हमारे पास एक single structure variable के लिए pointer भी हो सकता है, लेकिन इसका उपयोग ज्यादातर तब किया जाता है जब हम structure variable की array के साथ काम कर रहे होते हैं।
#include 
struct Book
{
char name[10];
int price;
}
int main()
{
struct Book a; //Single structure variable
struct Book* ptr; //Pointer of Structure type
ptr = &a;
struct Book b[10]; //Array of structure variables
struct Book* p; //Pointer of Structure type
p = &b; 
return 0;
}

Structures:-

Computer के terms में, एक data structure computer की memory में data को store और organize करने का एक Specific तरीका है ताकि इन data को बाद में efficiently उपयोग किया जा सके। data को कई अलग-अलग तरीकों से व्यवस्थित किया जा सकता है जैसे कि data के किसी particular organization के लिए logical या mathematical moodel को data structure कहा जाता है। एक specific data model की variety दो factors पर निर्भर करती है –
सबसे पहले, इसे real world object के साथ data के actual relationship को reflect करने के लिए structure में पर्याप्त load किया जाना चाहिए।
दूसरे, formation होना चाहिए ताकि कोई भी हर बार आवश्यक रूप से data को efficiently process कर सके।


Trees:-एक tree nodes नामक elements का एक collection है। प्रत्येक node में कुछ value या element होते हैं। हम binary tree के साथ vertex के बजाय node शब्द का उपयोग करेंगे। node किसी भी tree की structure का मुख्य component है। यह actual data को अन्य nodes के link के साथ store करता है।

Key properties of Tree:=A tree T is represented by nodes and edges, which includes:
1. T is empty (called null or empty tree).
2. T has a left subtree and right subtree.

Graph:-

graph एक non linear data structure होता है | इसका use कई application के लिए किया जाता है जैसे analysis of electrical circuit , finding shorted root , communication line , railway line etc graph G V तथा E होता है | 
यहाँ पर V = VERTEXहै तथा E=EDGE है|

Type of graph:-

1.DIRECTED GRAPH:-वह ग्राफ जिसमे EDGE की कोई दिशा होतीहै डायरेक्ट ग्राफ कहलाता है | और इस प्रकार के EDGE को DIRECTED EDGE कहते है || ग्राफ में edge को एक रेखा के through दर्शाया जाता है और यदि प्रत्येक रेखा में arrow का निशान बना हुआ होता है तो वह DIRECTED GRAPH कहलाता है | 


2.UNDIRECTED GRAPH:-वह ग्राफ जिसमे EDGE की कोई दिशा नहीं होतीहै| ग्राफ में edge को एक रेखा के through दर्शाया जाता है और यदि प्रत्येक रेखा में एरो का निशान नहीं बना हुआ होता है तो वह UNDIRECTED GRAPH कहलाता है |

3.WEIGHTED GRAPH AND NON WEIGHTED GRAPH:-कभी कभी graph में edges होतीहै | वे weight को carry करते है| ये weight वास्तविक नंबर होते है | directed और undirected graph दोनों ही weighted graph हो सकते है |

1.6 Mathematical notations and functions:-

1. Floor and Ceiling Functions:-If x is a real number, then it means that x lies between two integers which are called the floor and ceiling of x. i.e.
|_x_| is called the floor of x. It is the greatest integer that is not greater than x.
| x | is called the ceiling of x. It is the smallest integer that is not less than x.
If x is itself an integer, then |_x_| = | x |, otherwise |_x_| + 1 = | x | E.g.
|_3.14_| = 3, |_-8.5_| = -9, |_7_| = 7
| 3.14 |= 4, | -8.5 | = -8, | 7 |= 7


2. Remainder Function (Modular Arithmetic):-If k is any integer and M is a positive integer, then:
k (mod M)
gives the integer remainder when k is divided by M.
E.g.
25(mod 7) = 4
25(mod 5) = 0


3. Integer and Absolute Value Functions:-

If x is a real number, then integer function INT(x) will convert x into integer and the fractional part is removed.
E.g.
INT (3.14) = 3
INT (-8.5) = -8
The absolute function ABS(x) or | x | gives the absolute value of x i.e. it gives the positive value of x even if x is negative.
E.g.
ABS(-15) = 15 or ABS | -15| = 15
ABS(7) = 7 or ABS | 7 | = 7
ABS(-3.33) = 3.33 or ABS | -3.33 | = 3.33

5. Factorial Function:-n! denotes the product of the positive integers from 1 to n. n! is read as ‘n factorial’, i.e.
n! = 1 * 2 * 3 * ….. * (n-2) * (n-1) * n
E.g.
4! = 1 * 2 * 3 * 4 = 24
5! = 5 * 4! = 120
6. Permutations:-Let we have a set of n elements. A permutation of this set means the arrangement of the elements of the set in some order.
E.g.
Suppose the set contains a, b and c. The various permutations of these elements can be: abc, acb, bac, bca, cab, cba.
If there are n elements in the set then there will be n! permutations of those elements. It means if the set has 3 elements then there will be 3! = 1 * 2 * 3 = 6 permutations of the elements.

1.7 Asymptotic Notation:जब time और space के संदर्भ में किसी भी algorithm की complexity का Analysing करने की बात आती है, तो हम कभी भी algorithm के लिए आवश्यक time और space को define करने के लिए एक exact number प्रदान नहीं कर सकते हैं, इसके बजाय हम इसे कुछ standard notations का उपयोग करके व्यक्त करते हैं, जिसे Asymptotic notation भी कहा जाता है। जब हम किसी भी Algorithm का analyze करते हैं, तो हमें आम तौर पर execution के लिए आवश्यक समय की मात्रा का representation करने के लिए या computer द्वारा algorithm code की lines को चलाने के लिए आवश्यक समय, memory access की संख्या, comparision की संख्या, memory पर (occupying)कब्जा करने वाले temporary variables काrepresent करने के लिए एक formula मिलता है। अंतरिक्ष आदि इस सूत्र में अक्सर unimportant details होते हैं जो वास्तव में हमें चलने के समय के बारे में कुछ नहीं बताते हैं। एक उदाहरण लेते हैं, यदि कुछ एल्गोरिथ्म में T (n) = (n2 + 3n + 4) की time complexity है, जो एक द्विघात समीकरण है। N के बड़े मानों के लिए, n 2 भाग की तुलना में 3n + 4 भाग unimportant हो जाएगा।

For n = 1000, n2 will be 1000000 while 3n + 4 will be 3004.
इसके अलावा, जब हम दो algorithm के execution time की तुलना करते हैं तो heigh order की terms के constant cofficient भी neglected हो जाते हैं।
एक algorithm जो 200n2 का time लेता है, कुछ अन्य algorithm की तुलना में fast होगा जो n3 time लेता है, 200 से बड़े n के किसी भी value के लिए। चूंकि हम केवल function की वृद्धि के asymptotic behavior में रुचि रखते हैं, constant factor ignored भी किया जा सकता है।
Types of Asymptotic Notations:-We use three types of asymptotic notations to represent the growth of any algorithm, as input increases:
1.Big Theta (Θ)
2.Big Oh(O)
3.Big Omega (Ω)
