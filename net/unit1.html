1.3 Architecture and advantages of .NET framework:-.NET framework एक Software Development framework है जिसे Microsoft ने 2002 में विकसित किया था। .NET framework केवल Windows Operating System में ही run होता है।.net को डॉट नेट भी कहा जाता है।.NET एक ऐसा Framework है जिसमें कि Microsoft environment के अंदर web-based तथा windows-based Software (Application) को विकसित किया जाता है।.NET Framework में CLR(Common language Runtime) Framework की आत्मा की तरह कार्य करता है।.NET Framework में बहुत बड़ी class Library होती है जिसे framework class library(FCL) कहते है।.NET फ्रेमवर्क java language की तरह ही Object-Oriented है लेकिन यह platform independent नही है। अर्थात यह केवल windows platform में ही run होता है।.NET framework एक ऐसा फ्रेमवर्क है जो बहुत सारी भाषाओं को support करता है।.NET फ्रेमवर्क GUI(graphical user interface) उपलब्ध कराता है। 

.Net Framework Architecture .NET फ्रेमवर्क के मुख्य component निम्न है:-
1)CLR(common language runtime)
2)CTS(common type system)
3).NET framework class library(FCL)
4).NET languages.
1) CLR(common language runtime):-runtime एक ऐसा components है जो कंप्यूटर में एक विशेष प्रोग्रामिंग लैंग्वेज में लिखे प्रोग्राम्स को execute करने के लिए होना चाहिए। वह किसी भाषा के runtime प्रदान करता है, उस प्रोग्रामिंग लैंग्वेज में लिखे गए प्रोग्रेम द्वारा जरूरी basic features सफलतापूर्वक execute करने के लिए होती हैं। जब किसी प्रोग्राम को execute किया जाता है तब basic functionality प्राप्त करने के लिए प्रोग्रामिंग भाषा के runtime links की आवश्यकता है।
उदाहरण के लिए VB.Net प्रोग्राम को execute करने के लिए कंप्यूटर जिस पर प्रोग्राम execute किया जा रहा है उसमें VB.Net runtime installed होना चाहिए। विभिन्न प्रोग्रामिंग भाषाओं के लिए अलग-अलग runtime की आवश्यकता developers के लिए कार्य बन गया था वडेवलपर्स विभिन्न प्रोग्रामिंग भाषा में लिखे गए कोड को स्टोर नहीं कर सकते हैं|
.Net CLR कई प्रोग्रामिंग भाषा के लिए एक single runtime environment प्रदान करके इस समस्या का समाधान प्रदान करता है। CLR execution time पर code manages करता है code management CLR का basic principle है। वह code जो CLR environment द्वारा execute होने के बजाय executed होता है, OS द्वारा निष्पादित प्रबंधन कोड के रूप में कहा जाता है। वह कोड जिसे OS द्वारा निष्पादित किया जाता है और CLR द्वारा नहीं, को unmanaged code कहा जाता है।

2) CTS(common type system):-CTS ऐसे types के समूह को describe करता है जिसका कि सभी .NET लैंग्वेज में समान रूप से प्रयोग किया जाता है। उदाहरण के लिए:- C# में एक class library का प्रयोग किया जाता है और इस class library का प्रयोग VB.NET में भी किया जाता है क्योंकि यह class library, CTS के द्वारा define है।यह types, value types तथा reference types हो सकते है।value types को values के द्वारा pass किया जाता है जो कि stack में स्टोर रहते है।reference types को reference के द्वारा pass किया जाता है जो कि heap में स्टोर रहते है।

3) Framework class library(FCL):-.Net framework class library various namespace से बनी एक विशाल(huge) library है। Namespace में related classes का एक group होता है, interface structure जिनका उपयोग .net programing languages द्वारा किया जा सकता है।Namespace को उनकी functionality द्वारा परिभाषित किया जाता है, System Data namespace data bases तक पहुँचने के लिए functionality को सम्‍मिलित करता है। इस तरह के System data Sql client के रूप में,यह Sql server database और system data oledb को एक्‍सेस करने के लिए उपयोग किया जाता है, जो object linking और embedding databases के लिए उपयोग किया जाता है।

1.5 Object Oriented Feature:-Programming मे दो प्रकार के program paradigm का use किया जाता है। यह Procedure Oriented और Object oriented programming हैं। 
Features of Object Oriented Programming1.Object
2.Class
3.Data abstraction and encapsulation
4.Inheritance
5.Polymorphism
6.Message Passing
1.Object:-यह OOPs की basic run-time entity है। जो कि किसी object (person, place, bank account etc.) को represent करता है। object, class का variable है जो कि class को execute करता है और यह memory मे अन्य variables की तरह ही space लेता है।
2.Class:-Class एक user defined data type है जो data और code को contain करता है जो कि object द्वारा use किया जाता है। class एक structure है जो कि object कि working को define करता है। class को create करने के बाद उसके कई objects बनाए जा सकते हैं। इस प्रकार कहा जा सकता है कि class एक ही प्रकार के objects का collection है।
3.Data Abstraction and Encapsulation:-Data और Functions को एक साथ bind करना Encapsulation कहलाता है। यह class का सबसे important feature है। इसमे डाटा को class के बाहर access नहीं किया जा सकता है केवल class के functions ही इसे access कर सकते है। data abstraction बिना background process की details के data को input और output करने से है जिसमे यह Class के functions के द्वारा perform होता है।
4.Inheritance:-यह OOPs का एक महत्वपूर्ण feature है जो कि एक class को दूसरी class के features को access करने की facility provide करता है। इसमे एक class का object अन्य class की properties को भी access कर use कर सकता है। यह reuseability के feature को implement करता है जहां किसी class मे नए features कोadd करने के लिए नई class बनाकर उसमें पुरानी class के features को भी implement किया जाता है साथ ही नई class मे और codes भी जोड़ दिये जाते हैं।
5.Polymorphism:-यह भी OOPs का important concept है जो की एक से अधिक form को बनाकर उसे use करने की facility देता है। यह Function और operators दोनों के द्वारा perform किया जाता है जहां function name या operator वही रहता है पर arguments की संख्या या type या operands के type अलग होने पर अलग प्रकार से use होते है और task को perform करते हैं। जैसे + operator numerical addition और strings को जोड़ने के लिये use होता हैं।
6.Message Passing:-OOPs मे objects परस्पर communicate करते हैं। objects information को send और receive कर communication करते हैं। OOPs मे message passing function call के समय perform होता है। जिसमें information function argument के रूप में function के माध्यम से object मे input होता है और object कीmethod उसे process कर result generate का देती है।
1.6 Visual studio .NET:-कई लोग हमेशा Visual Studio .NET (VS.NET) और .NET technology से confused होते हैं। VS.NET सिर्फ एक संपादक(editior) है, जो Microsoft द्वारा developers को .NET प्रोग्राम आसानी से लिखने में मदद करता है। VS.NET editior automaticaly बहुत सारे code उत्पन्न करता है, developers को फॉर्म पर नियंत्रण को खींचने और छोड़ने की अनुमति देता है, application को compile करने औरे build के लिए शॉर्ट कट provides करता है।
.NET प्रोग्रामिंग करने के लिए VS.NET एक आवश्यक चीज नहीं है। आप अपने .NET कोड को लिखने के लिए बस एक नोटपैड या किसी अन्य simple editior का उपयोग कर सकते हैं। और आप कमांड प्रॉम्प्ट से अपने .NET प्रोग्राम को compile कर सकते हैं। ठीक है, हमने जो कहा वह सैद्धांतिक रूप से सही है लेकिन यदि आप .NET प्रोग्रामिंग के लिए नोटपैड का उपयोग करने का निर्णय लेते हैं, तब तक जब आप कुछ नमूना अनुप्रयोग विकसित करते हैं, तो Microsoft ने कुछ अन्य नई तकनीक पेश की होगी और .NET पुराना हो जाएगा। आप ऐसा नहीं चाह सकते हैं। तो, हमें VS.NET द्वारा चलते हैं, क्योंकि यह हमारे लिए चीजों को बहुत आसान बनाता है।
