6.1 Component of Data Warehouse:-

The data warehouse is based on an RDBMS server which is a central information repository that is surrounded by some key components to make the entire environment functional, manageable and accessible

There are mainly five components of Data Warehouse:

1. Data Warehouse Database

2. Sourcing, Acquisition, Clean-up and Transformation Tools (ETL)

3. Metadata

4. Data Mart

5. Query Tools

6.2 Meta data:-


6.2.1 Introduction:-meta data का प्रयोग वर्ड वाइड वेब(WWW) में सबसे ज्यादा प्रयोग किया जाता है। meta data वेब पेजों की जानकारी को contain करता है। meta data वेब पेज के keywords को भी स्टोर किये रहता है जिससे कि सर्च इंजन में वेब पेज के आने के chances बढ़ जाते है। Meta data शब्द का प्रयोग सन् 1968 में Philip Bagley ने किया था। meta data डेटा की सूचना को summary के रूप में contain किये रहता है जिससे कि हम उस डेटा को आसानी से सर्च कर सकते है तथा उसमें आसानी से कार्य कर सकते है।

6.2.2 Definition:-वह डाटा जो दूसरे डाटा को describe करता है मेटा डाटा कहलाता है अर्थात मेटा डाटा दूसरे डाटा के बारे में सुचना उपलब्ध करवाता है |

6.2.3 Types of Meta Data:-

1. Technical meta data:-

technical meta data में निम्म document include किये जाते ह|ैं

3. Extraction and Transformation Meta Data:-

4. End User Meta Data:-

यह data ware house का nevigation map है यह end user को business के बारे मे सोचने के तरीके की information देने मे मदद करता है 
उदाहरण के लिए:-कोई pdf book है जिसमें लेखक का नाम, उसका title, उसके छपने की तारीख तथा अन्य इनफार्मेशन दी हुई होती है जिससे कि उस किताब के बारें में आसानी से पता चलता है|

1. यह एक glue के रूप में work करता है जो data ware house के सभी cards को जोड़ता है|
2. यह building ,managing और data ware house use के लिए है|

2. Manager Query Tools:-

यह user data के बीच meta data layer डालकर समान और data dase संरचना पर जटिलता से चिल्लाता है |

5. Data mining tools:-

डाटा माइनिंग लार्ज अमाउंट के डाटा को माइनिंग करके सार्थक correlation ,pattern और trends की खोज की एक प्रोसेस है इस प्रोसेस को आटोमेटिक बनाने के लिए डाटा माइनिंग टूल का उपयोग किया जाता है 


6.5 Data Warehouse Databse:-central data warehouse डेटाबेस डेटा वेयरहाउसिंग environment की cornerstone है। यह data base लगभग हमेशा relational database management system (RDBMS) techinque पर लागू होता है। हालांकि, इस तरह के implementation को अक्सर इस fact से विवश किया जाता है कि traditional RDBMS product transactional database processing के लिए अनुकूलित हैं। कुछ डेटा वेयरहाउस विशेषताएँ, जैसे बहुत बड़े database size, adhoc query processing और aggregates, multi-table joins और drill-down सहित flexible users view निर्माण की आवश्यकता, डेटा warehouse database के विभिन्न technique approaches के लिए driver बन गए हैं। इन तरीकों में शामिल हैं|
1.scalability के लिए parallel relational database design जिसमें share-memory, share disk या विभिन्न mult-processor configuration (सममित) पर लागू किए गए साझा-कुछ model शामिल हैं (multiprocessor या SMP, बड़े पैमाने पर parallel या MPP, और / या uni- या multiprocessor के क्लस्टर)।
2.relational table scane को bypass करने के लिए नए index structure का उपयोग करके एक traditional RDBMS को गति देने के लिए एक innovative approch है ।
3.Multidimensional database (MDDBs) जो owner database technique पर based हैं; इसके विपरीत, एक dimensional data model एक familliar RDBMS का उपयोग करके लागू किया जा सकता है। multi-dimensional database को relational database model की प्रकृति द्वारा warehouse पर रखी गई किसी भी limit को पार करने के लिए design किया गया है। MDDBs on-line Analytical processing (OLAP)tool को सक्षम करते हैं, जो data query, repoting, analysis और mining tools के रूप में संयुक्त रूप से वर्गीकृत डेटा वेयरहाउसिंग घटकों के group से संबंधित हैं।
