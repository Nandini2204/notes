2.1 Introduction:-प्रोसेसिंग (Processing) शब्‍द को समझते हैं, कंप्यूटर (Computer) की भाषा में किसी भी काम जब किया जाता है तो उसमें काम को होने में शुरू से लेक‍र आखिर तक जो भी प्रक्रिया होती है वह प्रोसेसिंग (Processing) कहलाती है किसी सूचना को प्राप्‍त करने के लिये हम डाटा (Data) यानि आंकड़े इक्‍ठ्ठठा करते हैं और इस डाटा (Data) का अपनी जरूरत के हिसाब से विश्लेशण या डाटा प्रोसेसिंग करते हैं इस विश्लेशण यानि डाटा प्रोसेसिंग की प्रक्रिया में डाटा से अर्थपूर्ण तथ्य, अंक या सांख्यिकी डाटा प्राप्‍त होता है इसी अर्थपूर्ण डाटा काे सूचना कहते हैं जिसे उपयोग में लाया जाता है| 
2.2 Tasks of Data pre processing:-

Major Task of Data Pre- processing:-


1.Data cleaning
2.Data integration
3.Data reduction
4.Data Transformation
5.Data discretization
1.Data Cleaning:- data cleaning डेटा को इस तरह से clean करने की एक प्रक्रिया है जिससे data को आसानी से integrate किया जा सकता है|
2. Data Integration:- Data integration सभी डेटा को integrate / संयोजित करने की एक प्रक्रिया है।
3. Data Reduction:- डेटा की कमी बड़े डेटा को इस तरह से एक बार कम करने की प्रक्रिया है, जिससे डेटा को आसानी से और अधिक tranfomed किया जा सकता है।
4.Data Transformation:- Data transfomation, डेटा को एक reliable shape में बदलने की एक प्रक्रिया है।
5.Data Discretization:- इन कार्यों के पूरा होने के बाद, डेटा खनन के लिए तैयार है।
2.3 Data Cleaning:-

चित्र : Data Cleaning
• real world डेटा incomplete,noisy और incosistent हो जाते हैं।
• Data cleaning routines attempt to fill in missing values ,smooth out noise while identifying outliers,and correct inconsistences in the data.
• Data cleaning tasks include:
Fill missing values
1.Ignore the Tuple:- यह आमतौर पर तब किया जाता है जब class label missing होता है|
2.Fill the missing value manually:- यह दृष्टिकोण समय लेने वाला है और कई missing values के साथ एक large data set संभव नहीं है।
3.Use a global constant to fill the missing:- सभी missing attribute values को same constant द्वारा बदलें|
4.Use the attribute mean to fill the missing value:- किसी विशेषता के लिए missing value को बदलने के लिए किसी particular value का उपयोग करें।
Smoothed the following steps:-

Binning:-

Binning methods ने अपने “neighborhood”, यानी इसके आस-पास के मानों से परामर्श करके एक डेटा मान को सुचारू किया।

Regression:-

Data को किसी function के लिए fitt करके data को सुचारू किया जा सकता है, जैसे regression के साथ। linear regression में दो attributes (या चर) को फिट करने के लिए “best” लाइन ढूंढना शामिल है, ताकि दूसरे की भविष्यवाणी करने के लिए एक attribute का उपयोग किया जा सके। multiple linear रिग्रेशनिस linear regresssion का एक विस्तार है, जहां दो से अधिक attributes शामिल होती हैं और data एक multidimensional surface पर फिट होते हैं।

Clustering:-

clustring द्वारा outlayer का पता लगाया जा सकता है, जहां समान value समूहों में व्यवस्थित होते हैं, या “cluster”।
2.4 Data Integration:-

चित्र :Data Integration Data integration data pre-processing के चरणों में से एक है जिसमें विभिन्न sources में data को संयोजित करना और users को इन data के integrate view के साथ प्रदान करना शामिल है।
• यह कई data store (डेटा स्रोतों) से डेटा को merge करता है|
• It includes multiple database ,data cubes or flat files.
• metadata, correlation analysis, data conflict detection, और semetic heterogeneity contributetowards smooth data integration.
• Data integration के लिए मुख्य रूप से 2 प्रमुख दृष्टिकोण हैं – आमतौर पर “tight coupling” और “loose coupling approach” के रूप में जाना जाता है।

Tight Coupling:-

ETL-Extraction,Transformation और Loading process के माध्यम से data को single physical location में different sources से खींचा जाता है। single physical location data को query करने के लिए एक समान interface प्रदान करता है।
ETL layer sources से data को मैप करने में मदद करती है ताकि एक समान डेटा प्रदान किया जा सके|
इसे दृष्टिकोण को Tight Coupling कहा जाता है क्योंकि इस दृष्टिकोण में data को query के समय भौतिक भंडार के साथ कसकर जोड़ा जाता है।

Loose Coupling:-

यहां एक virtual mediated schema एक इंटरफ़ेस प्रदान करता है जो user से query लेता है, इसे उस तरीके से रूपांतरित करता है जिसे source database समझ सकता है और फिर परिणाम प्राप्त करने के लिए query को सीधे source database को भेजता है।
2.5 Data Transformation:-Data minnig के pre-processes और विशेष रूप से metadata और data warehouse में, हम data को destination data से स्रोत डेटा प्रारूप से परिवर्तित करने के लिए data transformation का उपयोग करते हैं।
• Data transformation में निम्नलिखित शामिल हो सकते हैं:
1. Smoothing:- Smoothing data cleaning process की तरह होती है | इसमें भी noisy data को remove करके data को clean किया जाता है | इसके लिए भी binning ,clustring और regression technique का use किया जाता है |
2. Aggregation:- Aggregation एक ऐसी technique है | जिसमे data पर aggregation operation करके data को summerized किया जाता है |
3. Generalization:- इस technique मे low -level डाटा को hierarchies concept का use करके high data से replaced किया जाता है | यह एक ऐसी process है | जिसमे data को generalize बनाया जाता है |
4. Normalization:- यह एक ऐसी process है | जिसमे database या table के contents को organise किया जाता है यह पर attributes data को sealed किया जाता है इसमें high range attributes को lower range data मे change किया जाता है | 
2.6 Data Reduction:-एक database या data warehouse डेटा की terabytes को स्टोर कर सकता है। इतनी बड़ी मात्रा में डेटा का analysis और mining करने में बहुत लंबा समय लग सकता है।
Data reduction techniques को data set के कम प्रतिनिधित्व को प्राप्त करने के लिए लागू किया जा सकता है जो volume में बहुत छोटा है लेकिन फिर भी महत्वपूर्ण जानकारी है।

Data reduction strategties:-

1.Data Cube Aggregation:- Data cube के निर्माण में data के लिए aggregation operations लागू होते हैं।
2.Dimensionality Reduction:- Dimensionality reduction में redundant का पता लगाया जाता है और हटाया जाता है जो data set के आकार को कम करता है।
3.Data Compreesion:- Data set आकार को कम करने के लिए encoding mechnisms का उपयोग किया जाता है।
4.Numerosity Reduction:- Numerosity reduction में जहां data को प्रतिस्थापित किया जाता है या वैकल्पिक द्वारा अनुमान लगाया जाता है।
5.Discretisation and Concept Hierarchy Generation:- जहां attributesके लिए range data value या higher conceptual levels द्वारा प्रतिस्थापित किया जाता है।
