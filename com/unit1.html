1.1.1 Framing:-framing दो computers या devices के बीच एक point to point connetion है जिसमें एक तार होता है जिसमें data bits की एक section के रूप में प्रसारित(aired) होता है। हालांकि, इन bits को जानकारी के discernible blocks में फंसाया जाना चाहिए। framing data link layer का एक function है। यह एक sender को bits के एक set को send करने का एक तरीका प्रदान करता है जो reciver के लिए meaningfull हैं। ethernet, token ring, frame relay और अन्य data link layer तकनीकों की अपनी frame structure हैं। frame में header होते हैं जिनमें error check code जैसी जानकारी होती है।
Data link layer पर, यह sender से message को extract करता है और इसे sender और reciver का address पता करके reciver को प्रदान करता है। frame का उपयोग करने का advantage यह है कि data recover करने योग्य chunks( विखंडू) में टूट जाता है जिसे आसानी से corruption(भ्रष्टाचार) के लिए जांचा जा सकता है। 

Framing में समस्याएं:–

1. Detecting start of the frame:- जब कोई frame प्रसारित(aired) होता है, तो हर station इसका पता लगाने में capable होना चाहिए। station bits के विशेष sequens की तलाश में frame का पता लगाते हैं जो fram की शुरुआत यानी SFD (स्टार्टिंग फ्रेम डेलिमीटर) को चिह्नित करता है। हर station क्रमिक circuit के माध्यम से SFD peturn के लिए link को सुनता है। यदि SFD का पता लगाया जाता है, तो अनुक्रमिक circuit alert station। fram को स्वीकार या अस्वीकार करने के लिए station check detection check करता है।
Detecting end of Frame:- frame को read करना कब बंद करना है। 

Framing के प्रकार:-

दो प्रकार के framing हैं:
1. Fixed size:- frame fixed size का होता है और frame को limitations प्रदान करने की कोई आवश्यकता नहीं होती है, frame की लंबाई ही सीमांकक( Delimiter) के रूप में कार्य करती है। दोष(defect): यह आंतरिक विखंडन से ग्रस्त है यदि data का आकार fram के आकार से कम है solution: Paddle|
2. Variable size:- इसमें fram के अंत को परिभाषित करने के साथ-साथ अगले fram की शुरुआत को भी अलग करने की आवश्यकता है।
इसे दो तरीकों से किया जा सकता है: 
Length field:- हम fram की लंबाई Point करने के लिए fram में एक लंबाई क्षेत्र का परिचय दे सकते हैं। ethernet (802.3) में प्रयुक्त। इसके साथ समस्या यह है कि कभी-कभी लंबाई field दूषित(Contaminated) हो सकती है। 
End delimiter(ED) – हम fram के end को point करने के लिए ED (पैटर्न) का परिचय दे सकते हैं। token ring में उपयोग किया जाता है। इसके साथ समस्या यह है कि ED data में हो सकता है। इसके द्वारा हल किया जा सकता है:
1. character/bit stafing: जब frams में character होते हैं तो उनका use किया जाता है। यदि data में ED होता है, तो byte को ED से अलग करने के लिए data में भरा जाता है। ED = “$” -> यदि data में कहीं भी anywhere $ ’है, तो इसे character \ O’ वर्ण का उपयोग करके बचाया जा सकता है। -> यदि डेटा में O \ O $ ‘है, तो \ \ O \ O \ O $’ का उपयोग करें (\ O का उपयोग करके बच जाता है और \ O का उपयोग करके बच जाता है)।
Disadvantage – यह बहुत costly और अप्रचलित (Obsolete) तरीका है। 2. bit stufing: ED = 01111 और यदि data = 01111 -> sender peturn को तोड़ने के लिए थोड़ा सा सामान करता है यानी यहाँ 0 को data = 011101 में जोड़ता है। -> reciver fram प्राप्त करता है। -> यदि data में 011101 है, तो reciver 0 को हटाता है और data को पढता है।

1.1.2 Error detection and correcting code:-

1.1.3 Error control:-Network एक device से दूसरे device में data के transmission के लिए जिम्मेदार है। transmitting application से data प्राप्त करने के अंत में प्राप्त करने वाले application में कई चरण होते हैं, प्रत्येक विषय में error। error control प्रक्रिया के साथ, हम आश्वस्त हो सकते हैं कि sent और प्राप्त data समान हैं। transmission के दौरान data दूषित हो सकता है। विश्वसनीय संचार के लिए, error का पता लगाया जाना चाहिए और उसे ठीक किया जाना चाहिए। error control bit level और packet level errors दोनों का पता लगाने और सही करने की प्रक्रिया है। त्रुटियों के प्रकार एकल bit error एकल bit error शब्द का अर्थ है कि data इकाई का केवल एक bit 1 से 0 और 0 से 1 में बदल दिया गया था। फटने की error term burst error का मतलब है कि data unit में दो या अधिक bits को बदल दिया गया था। burst error को packet level error भी कहा जाता है, जहाँ packet loss , duplication, reordaring जैसी errors होती हैं।


1.2 LAN PROTOCOL:-LAN या local area network network device को इस तरह से जोड़ता है कि personal computer और work station data, tools और program को साझा कर सकते हैं।TCP/IP protocol द्वारा परिभाषित एक निजी addressing योजना का उपयोग करके computer और devices के group को एक switch, या switch के ढेर से जोड़ा जाता है। स्थानीय network पर अन्य computerts के संबंध में निजी address अद्वितीय हैं।router एक LAN कीसीमा पर पाए जाते हैं, उन्हें बड़े WAN से जोड़ते हैं।
Data बहुत तेज़ दर पर प्रसारित होता है क्योंकि link कि एग एcomputers की संख्या सीमित होती है। परिभाषा के अनुसार, connection उच्चगति और अपेक्षाकृत सस्ते hardware (जैसे hub, network adepter और Ethernet cable) होना चाहिए। LAN छोटे भौगोलिक क्षेत्र को कवर करते हैं (आकार कुछ किलोमीटर तक सीमित है) और निजी स्वामित्व में हैं।एक office building, house, hospital, school, आदि के लिए इसका उपयोग कर सकता है।Lan design और रख-रखाव के लिए आसान है। LAN के लिए उपयोग कि जाने वाले एक संचार माध्यम में मुड़ जोड़ी केबल और समाक्षीय के बल होते हैं।यह एक छोटी दूरी तय करता है, और इसलिए error और noise कम हो जाता है।
4 से 16 mbps range में शुरुआती Lan की data rate थीं। आज, गति सामान्य रूप से 100 या 1000 mbps है। एक Lan में प्रचार विलंब बहुतकम है। सबसे छोटा LAN केवल दो computers का उपयोग कर सकता है, जब कि बड़ा LAN thousand computers को समायोजि तक रसकता है।एक LAN आमतौर पर बढ़ी हुई गति और सुरक्षा के लिए wired connection पर निर्भर करता है, लेकिन wireless connection भी LAN का हिस्सा हो सकता है। एक Lan की गलती सहनशीलता अधिक है और इस network में कम भीड़ है। उदाहरण के लिए: एक ही कमरे (internet के बिना) में counter straike खेलने वाले students का एक group।


1.2.1 ETHERNET AND IEEE 802.3 STANDARD CSMA/CD:-Ethernet को “ether net” उचारण किया ज्याता है. ये एक Local Area Network Technology है. इस Technology की मदद से Computers और Networking Devices को आपस में connect किया ज्याता है और information को share किया ज्याता है. जैसे office में, College में, School में किया ज्याता है. “Ethernet” TCP/IP Stack के data link layer का Protocol है. इस Ethernet Technology की मदद से ही LAN में अलग अलग Computer आपस में Information share कर पाते हैं. ये Protocol मतलब Ethernet का काम है, Information किस Format में Transmit होगा. जैसे की वो एक LAN में information अछे से दुसरे Computer तक पोहांचा सके बिना किसी Error के.Ethernet LAN में Networking devices के बिच में Information का Communication करवाता है. आप को इसको को अछेसे समझाने के लिए एक आसन सा उदहारण लेते हैं. जैसे आपका Computer LAB. जहाँ पे Ethernet cable मतलब twisted pair cable की मदद से ही सारे Computer आपस में connected होते हैं. लेकिन जो Information आपके पास आपके Computer तक पोहंचती है, उस Technology का नाम है Ethernet. जिसको Ethernet protocol भी बोला ज्याता है. वैसे आपको ये तो पता होगा OSI Network Model के 7 layers होते हैं उनमे से “Ethernet” data link layer और “physical layer “दोनों में काम करता है.
Ethernet data Transmission में दो तरह के यूनिट का इस्तेमाल करता है पहला Frame और दूसरा Packet (जैसे आप चावल को Packet में भी तोलते हो, आप इससे किलो ग्राम में, और टन में भी तोलते है ऐसे ही data Frame और Packet दोनों रूप में network से गुजर ता है). Frame केवल Payload को लेके नहीं ज्याता बल्कि वो MAC Address को भी साथ में लेके ज्याता है. MAC address Computer का address होता है, जैसे की वो Sender और Receiver Computers का पता प्राप्त कर सके.

रिंग इंटरफेस के दो ऑपरेटिंग मोड हैं।सुनते हैं और प्रसारित करते हैं।सुनने के मोड में, इनपुट बिट्स को केवल 1- बिट समय की देरी से आउटपुट में कॉपी किया जाता है।संचारित मोड में इनपुट और आउटपुट के बीच का कनेक्शन इंटरफ़ेस से टूट जाता है, जिससे वह अपना डेटा सम्मिलितकर सकता है।जब यह टोकन कैप्चर करता है, तो स्टेशन ट्रांसमिशन मोड में आता है।
फ़्रेम को बहुत सरल तरीके से गंतव्य द्वारा स्वीकार किया जाता है।प्रेषक, ACK बिट के साथ रिसीवर को फ़्रेम भेजता है।फ़्रेम प्राप्त करने पर रिसीवर, अपने बफर में डेटा की प्रतिलिपि बनाता है, चेक सम को सत्यापित करता है और ACK बिट को 1 पर सेट करताहै।सत्यापित फ़्रेम वापस प्रेषक के पास आते हैं,जहाँ उन्हें रिंग से निकाल दिया जाता है।
सूचना फ़्रेम रिंग को तब तक प्रसारित करता है जब तक कि वह इच्छित गंतव्य स्टेशन तक नहीं पहुंच जाता है,जो आगे की प्रक्रिया के लिए जानकारी की प्रतिलिपि बनाता है।सूचना फ्रेम रिंग को गोल करना जारी रखता है और अंत मे ंइसे भेजने वाले स्टेशन तक पहुंचने पर हटा दिया जाता है।भेजने वाला स्टेशन लौटने के फ्रेम की जांच कर सकता है कि क्या फ्रेम को देखा गया था और बाद में गंतव्य द्वारा कॉपी किया गया था। एक स्टेशन समय की एक विशिष्ट अवधि के लिए एक टोकन धारण कर सकता है।इस समय के दौरान,इसे अपने प्रसारण को पूरा करना होगा और रिंग में टोकन को पुन:उत्पन्न करना होगा।जब भी कोई स्टेशन अपना प्रसारण खत्म करता है,तो दूसरा स्टेशन टोकन पकड़ लेता है और अपना प्रसारण शुरू कर देताहै।
रिंग नेटवर्क में केबल का टूटना फोरस्टेशन कनेक्टेड वाया एवायर सेंटर
• यदि केबल टूट जाता है, तो पूरा रिंग नेटवर्क नीचे चला जाता है।यह रिंग में टोकन के प्रसार को पूरी तरह से रोक सकता है।
• इस समस्या को तार केंद्र का उपयोग करके हल किया जा सकता है जैसा कि अंजीर में दिखाया गया है।
• यह वायर सेंटर उस टर्मिनल को बाय पास करता है जो निम्नलिखित तरीके से नीचे चला गया है:-
(ए) प्रत्येक स्टेशन तार केंद से दो मुड़ जोड़े युक्त होता है, एक स्टेशन से डेटा के लिए और दूसरा स्टेशन से डेटा के लिए।
(b) वायर सेंटर के अंदर स्टेशनेरंट द्वारा सक्रिय होने वाले बाई पास रिले हैं|
(c) यदि रिंग टूट जाती है या कोई स्टेशन चला जाता है तो ड्राइव करंट का नुकसान होता है और रिले स्टेशन को छोड़ देगा।


1.3 PPP(POINT-TO-POINT PROTOCOL):-PPP का पूरा नाम point to point protocol है | यह कंप्यूटर नेटवर्क प्रोटोकॉल है जिसका प्रयोग multi-protocol datagram point-point links माध्यम transmit करने के लिए किया जाता है | 
अर्थात, इसका प्रयोग दो direct connected computer के मध्य datagram को transfer करने के लिए किया जाता है | 
PPP को IETF (इंटरनेट इंजीनियरिंग टास्क फाॅर्स) के द्वारा बनाया गया था | इसे बनाने का मुख्य उद्देश्य point to point links के लिए data link protocol create करना था | point to point protocol सबसे ज्यादा प्रयोग किये जाने वाला limk protocol है इसका प्रयोग modam के द्वारा computer को internet service provider (ISP) के server से connect करने के लिए किया जाता है | PPP बहुत सी सुविधाएँ प्रदान करता है जो निम्न है –
1.यह तथा की सुविधा है | 
2.यह डिवाइसों के मध्य ट्रांसमिट होने वाले frames के format को डिफाइन करता है |
3 यह लिंक कण्ट्रोल प्रोटोकॉल(LCP) को डिफाइन करता हे | . 
4. यह नेटवर्क लेयर डाटा केencapsulation को define करता है | 
5. यह auto-configuration की सुविधा देता है | 
PPP framing:- यह एक byte-oriented protocol है इसमें framing byte-oriented protocol के हिसाब से होती है | 
PPP frame format में निम्न fields होते है-
1.FLAG field:- एक PPP frame 1 byte के साथ start भी होता है और end भी होता है|
2.address:- इस protocol में address field हमेशा नियत रहता है इसकी वैल्यू हमेशा 11111111 रहती है | 
3. control field:-यह field एक नियत वैल्यू 11000000 पर सेट रहता है PPP flow कण्ट्रोल की प्रदान नहीं करती है इसलिए इस field कि जरूरत नहीं पड़ती है| 
4. protocol field:- protocol field यह डिफाइन करता है कि डाटा फील्ड प्रकार का डाटा है| 
5. data field:-यह एक length variable है यह user data तथा अन्य सुचना को स्टोर करता है | 
6. FCS field:- यह frame check sequence है यह 2 बाइट या 4 बाइट स्टैण्डर्ड कर्क होता है यह checksum को conatin किये रहता है|


1.4 FDDI(FIBER DISTRIBUTED DATA INTERCONNECT):-FDDI (फाइबर डिस्ट्रिब्यूटेड डेटा इंटरफेस) एक स्थानीय क्षेत्र network (LAN) में फाइबर ऑप्टिक लाइनों पर data transmission के लिए ANSI और ISO मानकों का एक set है जो 200 km (124 मील) तक की सीमा में विस्तारित होता है। FDDI प्रोटोकॉल टोकन रिंग प्रोटोकॉल पर आधारित है। भौगोलिक रूप से होने के अलावा, एक FDDI स्थानीय क्षेत्र नेटवर्क हजारों उपयोगकर्ताओं का समर्थन कर सकता है। एफडीडीआई का उपयोग अक्सर एक विस्तृत क्षेत्र नेटवर्क (WAN) के लिए back bone पर किया जाता है|
FDDI network में दो token ring होते हैं, प्राथमिक रिंग विफल होने की स्थिति में संभावित backup के लिए एक। प्राइमरी रिंग में 100 mbps तक की क्षमता है। यदि backup के लिए द्वितीयक रिंग की आवश्यकता नहीं है, तो यह डेटा ले जा सकता है, 200 mbps तक की क्षमता का विस्तार single ring अधिकतम दूरी का विस्तार कर सकता है; एक दोहरी रिंग 100 km (62 मील) का विस्तार कर सकती है।
FDDI अमेरिकी राष्ट्रीय मानक समिति X3-T9 का एक उत्पाद है और open system interconnection (OSI) कार्यात्मक layering के modale के अनुरूप है। अन्य प्रोटोकॉल का उपयोग कर LANs। FDDI- II FDDI का एक संस्करण है जो नेटवर्क में सर्किट-स्विच्ड सेवा को जोड़ने की क्षमता जोड़ता है ताकि वॉइस सिग्नल को भी हैंडल किया जा सके। FDDI नेटवर्क को विकासशील सिंक्रोनस ऑप्टिकल नेटवर्क (SONET) से जोड़ने के लिए काम चल रहा है।
