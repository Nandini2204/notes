1. Upward multiplexing:-

यह single layer के connections को एक single lower layer के connection पर ही multiplex करती है| चार विभिन्न transport connection एक ही network connection का use कर अपने remote host पर communicate करते है| 
यह multiplexing की approach connection की अवधि पर निर्भर करती है|transport layer विभिन्न transport connections को उनके destination के अनुसार विभिन्न समूहों में विभाजित करती है,तत्पश्चात उन समूहों को न्यूनतम network connection पर map किया जाता है| अब यदि अत्यधिक transport connection को एक network connection की performance को प्रभावित करेगा और यदि अत्यधिक कम connection को एक network layer पर map किया जायेगा तो यह connection को महंगा बना देगा,जिसमे कि connection के द्वारा resources का westage होगा | 

2. Downward multiplexing:-

downward multiplexing के अंतर्गत transport layer के एक connection को lower layer पर split किया जाता है| Transport layer कई network connection बनाती है व के transport connection को इन multiple network connection पर map कर देती है| यह approach traffic को विभिन्न connection पर round robin manner में बाँटता है| Network connection bandwidth को हम factor 'n' से बढ़ा देते है| साथ ही यह performance को बढ़ाता है|

2. Syn remote transport entity को confirmation के लिए SYN भेजा जाता है| 
3. Connection को establish state में रखा जाता है| 
इसमें two way handshake में connection establish करना बताया गया है| कोई भी host connection establish करना start कर सकता है| यदि दोनों ही host connection establish करना चाहते है तो connection बिना रुकावट के establish हो जाता है| इसे स्थायी रूप से terminate करने के लिए कोई भी side close command का use कर सकता है| 

Problems with two way handshake:-

two way handshake की main problem lost हुए syn व ack है| इसमें होस्ट A,B को connection establish करने के लिए syn भेजता है और hope रखता है| B से की वह उसका ack भेजेगा| इसमें दो condition create होती है-
1. A द्वारा भेजा गया syn lost हो जाये| 
2. B द्वारा भेजा गया ack lost हो जाये 
दोनों ही condition में syn timer को retransmit करता है| यह duplicate syn को जन्म देती है| यदि B का syn lost हो जाये तो वह दो syn receive करेगा| और यदि B का ack lost न होकर delay हो जाये तो वह A दो ack receive करेगा| 
जब तक host B two way handshake की अन्य problem obselete syn segment से ग्रसित रहता है| इसके अंतर्गत किसी पुराने connection का syn timer out हो जाने के बाद B इसे एक new request मानते हुए इसका ack भेज देती है| इसके मध्य यदि A वापस connection establish करना चाहता है तो वह syn भेज देता है| B इसे duplicate मानकर reject कर देता है| A connection establish मानकर data transfer करता है| तो B इसे invalid मानते हुए reject कर देता है| इस problem को दूर करने के लिए three way handshake का use किया जाता है| जिसमे दोनों side अलग connection की request को acknowledge करता है और sequence no भी generate करती है|3.2.4 Releasing a connetion :-Releasing a connection transport layer का main part होता है| इसमें 2 device के मध्य connection release किया जाता है| *Releasing a connection उसके establishing से easy process है| *Releasing a connection में जो data last में भेजा जाता है वह successfully अपने destination तक पहुंचना चाहिए व receive होना चाहिए| *Terminal process को graceful तरीके से use करने के लिए दोनों side final data segment को explicitly acknowledge करना चाहिए|

3.2.5 Reliable delivery:/delivery control(segments):--ट्रांसपोर्ट लेयर पर reliable data deliavery निम्न प्रकार के aspects रखती है जिनमे error control , sequence control loss control व duplication control प्रमुख है|

(i)error control:-

-data transmission के दौरान उसका प्राथमिक उद्देश्य data की reliability को बनाये रखना है इस हेतु इसमें error control महत्वपूर्ण बिंदु है | 
-real मे 100% data carefully transfer करना impossible परन्तु transport layer को इसके nearest condition अनुरूप design करने के प्रयास किये गए है|
-error control मे error detection व रट्रान्समिशनपर आधारित है | जो की error control handling के लिए अलग-अलग algorithm का use करती है | जैसे की checksum 
-data link layer मे node-to-node error free delivery की गारंटी तो होती है | परन्तु node-to-node reliability end-to-end reliability भी दे यह आवश्यक नहीं है |

(ii) sequence control:-

-transport layer मे reliability बनाये रखने के लिए एक महत्वपूर्ण aspect sequence control है | 
-हमें ज्ञात है की all data एक packet में नहीं भेजा जाकर अलग-अलग sequence packet में भेजा जाता है| 
-sequence control का मुख्य कार्य यह होता है | की sending end पर upper layer से receive किये जनर वाली data unit lower layer के लिए useful हो तथा साथ receiving end परयह विभिन्न packets या data units सही प्रकार से पुनः एकत्रित हो सके|

(iii) segmentation व concatenation:-

-जब upper layer से received data units का size network layer datagram या data link layer के frame size से ज्यादा ही बड़ा होता है | तब transport layer protocol इन data units को छोटे-छोटे ;useful blocks मे तोड़ देते है | यह dividing process ही segmentation कहलाता है | 
-जब lower layer को upper layer से received data units का size एक session के लिए भी उचित नहीं है| तब कई data inits को मिलाकर datagram या frame बनाया जाता है| यह कार्य transport लेयर protocol के द्वारा किया जाता है | combining का यह process concatenation कहलाता है| 

(iv)Sequence number:-

-प्रत्येक transport layer service के अंतर्गत प्रत्येक segment के अंत मे उसका segment नंबर उपरोक्त service के through attach किया जाता है | 
-यदि बड़ी संख्या मे data unit kosegment मे collect कर send किया जाता है | तो यह sequence number उसे पुनः reassemble करने मे सहायक होते है | 
-प्रत्येक segment के अंतर्गत एक फील्ड ऐसा होता है जो यह बताता है को यह segment अंतिम है या मध्यम का है | 
-sender व receiver के according यह important है की किसी भी order से transmit होने वाले data units अपने destination पर proper reassemble हो जैसे की डाटा transmission से पूर्व source end पर थे |

(v)loss control:-

यह transport layer का एक महत्वपूर्ण reliability aspect है जिसके अंतर्गत यह transport layer सुनिश्चित करती है की destination end पर भी वह सभी डाटा units पहुंची है जो की ट्रांसमिट की गई 
-data को segment form मे transfer करते समय segment के lost होने का डर रहता है | जिसे हम receiver का transport layer protocol sequence number की help से missing lost segment को identify कर पुनः delivery की request sender को send करते है | 

(vi) duplication control:-

transport layer द्वारा data की raliability को बनाये रखने लिए duplication control का 6th aspect है|
-किसी भीं data segment duplication form मे होना| यदि ऐसा तो इसकी पहचान sequence number के through को जाती है | 
-duplication data को identify कर उसे discard कर दिया जाता है | अन्यथा यह system के लिए हानिकारक हो सकती है |
3.2.6 Flow control and buffering:-data link layer की अपेक्षा transport layer में flow control अधिक जटिल प्रक्रिया है| 
- Flow control वह monitoring है जो कि प्रत्येक connection के receiver पर भी तीव्रता से data को transmit कर सके| 
-Transport layer में data transmission का कार्य buffering का use करके किया जाता है| 
-इसके अंतर्गत buffer का size व उसका space दो host द्वारा data के transmission से पहले specify किया जाता है|
-अर्थात buffer कितने data को store कर सकेगा व उसका size TPDU size द्वारा निर्धारित होगा| 

Types of buffering:-

1. Fixed size buffer:-
-प्रत्येक buffer का size equal होता है| 
- यह एक ideal buffering scheme है जिसमे प्रत्येक TPDU को प्रत्येक buffer store किया जाता है|
2. Variable size buffer:- -buffer में विभिन्न size के buffer होते है| 
-प्रत्येक TPDU को store किया जाता है| 
-यह larger buffer maximum size वाले TPDU को ध्यान में रखकर बनाया जाता है|
-यदि सभी TPDU छोटे होते है तो space wasted होता है| 
- source buffering व destination buffering के मध्य का trade off दोनों transport layer connection के आने वाले traffic पर निर्धारित होता है| 
- दूसरी तरफ flow control को अधिक सुचारु करने के लिए Receiving transport entity fixed sliding window protocol व एक credit scheme का use करती है| 
- sliding window के तीन प्रमुख issues है-
(a) यह data segment पर sequence number का use कर|
(b) यह fixed size windows का use करती है| 
(c) windows को forward करने के लिए acknowledgement का use|
- sliding window technique reliable network service में successful कार्य करती है,but unreliable network service में यह सफल नहीं है क्योकि वहाँ segment lost भी हो सकते है|

(b) ACK:- यह field acknowledgement number field के valid होने की दशा में set किया जाता है|
- ACK की absence में इसे '0' कर ignore कर दिया जाता है| 
(c) PSH:- यह push function को बताती है, जो कि पुरे data segment की delivery से related है, जब तक कि full buffers receive नहीं हो जाता है| 
(d)RST:-यह किन्ही कारणों से crash हुए connection को पुनः reset करने के लिए उपयोग में आने वाली bit है| 
(e) SYN:-यह bit connection को establish करने के काम में आती है| 
- यह SYN bit connection के establishment request के लिए व उसके accept होने पर ack bit का use किया जाता है| 
(f) FIN:-यह bit connection के release होने की दशा में use में आती है| 
- यह sender की ओर data transmit करने से रोकती है|

3.4.2 TCP Connection management:-- TCP connection 2 way handshake के स्थान पर 3 way handshake mechanism का use करता है| 
- यह full duplex mode में data transfer करता है| 
-किसी भी host system में जब दो TCP module connect किये जाते है तो वे एक दूसरे को एक साथ segment send करने में capable हो सकते है|
-इसका अर्थ यह है कि data transfer करने से पूर्व प्रत्येक host प्रत्येक process को communicate करना पड़ता है और दूसरे process की और से ok signal प्राप्त करना पड़ता है|
-Data send करने के पहले निम्न 4 process से pass होना पड़ता है-
(1) Host A connection के लिए अपनी इच्छा घोसित करते हुए एक segment भेजता है तथा A से B को traffic के बारे में अपनी initialization information include करती है|
(2) A की request को acknowledge करने के लिए Host B एक segment sent करता है| 
(3) इस segment में पुनः B से A को traffic की initialization information include करती है| 
(4) B की request को acknowledge करने के लिए Host A एक segment send करता है| 
- यहाँ last step complete होने पर connection establish कर दिया जाता है|
- TCP एक 3-way handshake technique का use करते है| 
3.4.3 Congestion control management:-- Congestion से तात्पर्य होता है network channel के द्वारा ले जा सकने वाले load से दिया जाने वाला load अधिक होगा| 
- प्रत्येक network के अंतर्गत उसके विभिन्न layers के मध्य data units को transmit करने के लिए आवश्यक channel की क्षमता निर्धारित होती है|
- इस क्षमता अनुरूप ही इन channel पर data units transmit किये जाते है परन्तु जब channel पर क्षमता से अधिक data units को transmit किया जाता है तो यह overload की स्तिथि ही congestion कहलाती है| 
- Congestion को सर्वप्रथम data link layer पर फिर network layer उसके बाहः transport layer पर नियंत्रित किया जाता है इसके लिए यह layers कई प्रकार की policy व strategy बनाती है| 
- Congestion control का सामान्य सिद्धांत control system theory पर आधारित है| 
- TCP/IP में data के overflow condition को control करने के लिए TCP में internet congestion control algorithm का use किया जाता है| 
- इस algorithm के अंतर्गत new parameter 'Threshold' का use करते है| 
- इस algorithm में starting में data segment के size को exponentially बढ़ाया जाता है|
- जब time out होने लगता है तो threshold को current congestion window में अधिक size पर set कर उसके बाद linear form में data segment के size बढ़ते हुए भेजे जाते है| 
3.4.4 Tcp timer management:-TCP में 7 प्रकार के timer management को maintain किया जाता है-
1. SYNACK:-हमारे SYN से ACK के लिए wait करना| 
2. RETRANSMIT:-data retransmission के लिए exponential back off रहता है| 
3. DELAY ACK:-यह piggy-back ACK के लिए hope रहती है| 
4. KEEP ALIVE:-यह connection को check करता है connection alive है या नहीं| 
5. PROBE:-यह test करता है कि zero window size still applies है या नहीं| 
6. FIN_WAIT_2:-यह closed के साथ switch होता है यदि no FIN received. 
7. TWKILL:-TIME_WAIT में stay रहने के लिए कितना लम्बा समय लगा|
