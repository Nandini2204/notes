4.1 principle of application layer protocol:-नेटवर्क एप्लिकेशन एक कंप्यूटर नेटवर्क के डीटेल हैं। हम अगर किसी भी उपयोगी अनुप्रयोगों के बारे में गर्भ धारण नहीं कर सकते, इसके लिए कोई आवश्यकता नहीं होगी उनका समर्थन करने के लिए नेटवर्किंग प्रोटोकॉल डिज़ाइन करें। लेकिन पिछले तीस वर्षों में, कई लोगों ने कई सरल और अद्भुत नेटवर्किंग तैयार की हैं अनुप्रयोगों। इन अनुप्रयोगों में क्लासिक पाठ-आधारित अनुप्रयोग शामिल हैं 1980 के दशक में यह लोकप्रिय हो गया, जिसमें कंप्यूटरों की रिमोट एक्सेस शामिल है, इलेक्ट्रॉनिक मेल, फ़ाइल स्थानांतरण, समाचार समूह और चैट। लेकिन वे भी शामिल हैं अभी हाल ही में वर्ल्ड वाइड जैसे मल्टीमीडिया अनुप्रयोगों की कल्पना की गई है वेब, इंटरनेट टेलीफोनी, वीडियो कॉन्फ्रेंसिंग, और ऑडियो और वीडियो पर मांग। हालांकि नेटवर्क एप्लिकेशन विविध हैं और कई परस्पर क्रिया करते हैं घटकों, सॉफ्टवेयर लगभग हमेशा उनके मूल में है। खंड से याद करें 1.2 कि एक नेटवर्क एप्लिकेशन का सॉफ्टवेयर दो या अधिक के बीच वितरित किया गया है एंड सिस्टम (यानी, होस्ट कंप्यूटर)। उदाहरण के लिए, वेब के साथ हैं सॉफ्टवेयर के दो टुकड़े जो एक दूसरे के साथ संवाद करते हैं: ब्राउज़र उपयोगकर्ता के मेजबान (पीसी, मैक, या वर्कस्टेशन) और वेब सर्वर में सॉफ्टवेयर वेब सर्वर में सॉफ्टवेयर। टेलनेट के साथ, फिर से दो टुकड़े हैं दो मेजबान में सॉफ्टवेयर: स्थानीय होस्ट में सॉफ्टवेयर और रिमोट में सॉफ्टवेयर मेज़बान। मल्टीपार्टी वीडियो कॉन्फ्रेंसिंग के साथ, प्रत्येक में एक सॉफ्टवेयर पीस है मेजबान जो सम्मेलन में भाग लेता है। ऑपरेटिंग सिस्टम के शब्दजाल में, यह वास्तव में सॉफ्टवेयर टुकड़े नहीं है (अर्थात, प्रोग्राम) जो संचार कर रहे हैं लेकिन सत्य प्रक्रियाओं में हैं संवाद। एक प्रक्रिया को एक प्रोग्राम के रूप में सोचा जा सकता है जो चल रहा है 
एक अंत प्रणाली के भीतर। जब संचार प्रक्रियाएं चल रही हों एक ही अंत प्रणाली, वे एक-दूसरे के साथ इंटरप्रोसेस का उपयोग करके संवाद करते हैं संचार। इंटरप्रोसेस संचार के लिए नियमों द्वारा नियंत्रित किया जाता है एंड सिस्टम का ऑपरेटिंग सिस्टम लेकिन इस किताब में हमें कोई दिलचस्पी नहीं है एक ही मेजबान पर प्रक्रिया कैसे संवाद करती है, लेकिन इसके बजाय कैसे विभिन्न अंत प्रणालियों पर चलने वाली प्रक्रियाएं (संभावित रूप से भिन्न के साथ) ऑपरेटिंग सिस्टम) संचार। दो अलग-अलग अंत प्रणालियों पर प्रक्रियाएं संदेशों का आदान-प्रदान करके एक दूसरे के साथ संवाद करें कंप्यूटर नेटवर्क। एक भेजने की प्रक्रिया बनाता है और संदेश भेजता है जाल; एक प्राप्त करने की प्रक्रिया इन संदेशों को प्राप्त करती है और संभवतः संदेश वापस भेजकर प्रतिक्रिया करता है,
नेटवर्किंग अनुप्रयोगों में अनुप्रयोग-परत प्रोटोकॉल होते हैं जो प्रारूप को परिभाषित करते हैं और संदेशों का क्रम प्रक्रियाओं के बीच आदान-प्रदान, साथ ही परिभाषित करता है संदेश के प्रसारण या प्राप्ति पर की जाने वाली क्रियाएं।एप्लिकेशन परत एक विशेष रूप से अच्छी जगह है प्रोटोकॉल। यह परिचित जमीन है। हम इनमें से कई से परिचित हैं ऐसे अनुप्रयोग जो प्रोटोकॉल पर निर्भर करते हैं, हम उनका अध्ययन करेंगे। यह हमें एक अच्छा देगा क्या प्रोटोकॉल के बारे में महसूस कर रहे हैं और हमें के कई के लिए मिलवा देंगे जब हम परिवहन, नेटवर्क, और डेटा-लिंक लेयर प्रोटोकॉल।


4.2 Doman name system (DNS):-Domain Name System को short मे DNS भी कहते है |DNS का काम Domain Name को IP Address मे convert करना होता है | जब हम किसी भी domain name को web browser मे टाइप करते है तो ये DNS उसको IP Address मे convert करता है | इसी प्रकार किसी भी Network मे किसी भी Computer एवं host name को भी ये IP Address मे कन्वर्ट करता है| यह इसलिए होता है क्योकि हम IP Address की तुलना मे Alphanumeric Name easily याद रख सकते है |एक उदहारण की लिए – जब हम http://www.example.com को browser मे type करते है तो DNS इसको 198.15.45.18 मे या इसी प्रकार की किसी valid IP Address मे चेंज कर देता है| अब आप खुद देखिए हमको http://WWW.example.com याद करना कितना आसान है जबकि IP address याद करना कितना मुश्किल| 
अब हम जानते है की Public IP Address के case मे अन गिनत DNS और IP Address हो सकते है तो केवल एक सर्वर सभी की information को save करके नहीं रख सकता, इसी वजह से जब query एक सर्वर के पास आती है एवं उस सर्वर को IP Address नहीं पता होता या वो उस query को resolve नहीं कर पता तो वो query को उससे connected दूसरे सर्वर के पास भेज देता है| यह Pass on तब तक चलता है जब तक query resolve नहीं हो जाती एवं IP Address नहीं मिल जाता|

Working of DNS:-

1. सबसे पहले user किसी website का URL web browser के address bar में लिखता है और enter press करता है| 
2. Browser सबसे पहले अपने cache को check करता है कि क्या उसके पास इस domain name का IP address पहले से मौजूद है| जब भी हम किसी website को visit करते है तो उसका IP address browser अपने cache में कुछ दिनों के लिए store कर लेता है| ऐसा इसलिए करता है कि ताकि same process को बार बार न दोहराया जाये और server पर load भी न पड़े| 
3. यदि browser को cache में ही IP address मिल जाता है तो user को उस IP address से connect कर दिया जाता है और website load हो जाती है| 
4. यदि browser को IP address अपने cache में नहीं मिलता है तो browser ये request OS को transfer करता है| OS इस request को resolver को transfer करते है| 
5. Resolver इस domain name के IP address के लिए खुद के cache को check करता है| यदि IP address मिल जाता है तो resolver इसे OS को और OS web browser को transfer कर देता है| 
6. यदि resolver को IP address नहीं मिलता तो वह इस request को root server तक पहुँचाता है| एक resolver आपका ISP (internet service provider) होता है| सभी resolver को root server के बारे में जानकारी होती है| 
7. Root server इस request को top level domain server को pass कर देता है| Top level domain server को com,gov,edu,org,in आदि servers के बारे में जानकारी होती है| इसलिए वह इस request को appropriate top level domain server को pass कर देता है| 
8. Top level domain server इस request को authoritative name server को pass कर देता है अभी तक की process में com server को locate किया जा चूका है,अब actual name server को locate किया जायेगा| जब भी कोई domain register की जाती है तो domain register उस domain name को store कर और top level domain servers के साथ इस information को share करता है| इसलिए top level domain server को पता रहता है कि actual name server कहाँ है| 
9. Authoritative name server request की गयी domain name का IP address provide करता है| Resolver इस IP address को अपने cache में store कर लेता है| साथ ही OS भी इस IP address को further request को serve करने के लिए अपने cache में store कर लेता है| 
10. आखिर में resolver IP address के base पर website को browser में load करता है| 


4.3 The file transfer protocol (FTP):-FTP की full form File Transfer Protocol होती है| यह एक application layer protocol है जो एक local file system से remote file system के मध्य files transfer करता है| इसके लिए एक FTP client व FTP server की आवश्यकता होती है| 
FTP user को इस योग्य बनाता है कि वह remote system में login कर सके| जब user login कर चूका होता है तब वह उस system पर directory की जाँच कर सकता है, files में changes कर सकता है,commands को execute कर सकता है और साथ ही कुछ अन्य कार्य भी कर सकता है| 
FTP एक client-server प्रोटोकॉल है इसलिए यह client और server के मध्य डेटा ट्रान्सफर करने के लिए दो TCP connections स्थापित करता है। FTP द्वारा एक TCP connection का प्रयोग conservation को नियंत्रित करने के लिए किया जाता है तथा दूसरे TCP connection का प्रयोग डेटा files को ट्रान्सफर करने के लिए किया जाता है।
सर्वप्रथम user FTP user agent की सहायता से Remote System पर login करता है जिसके लिए आवश्यक user name व password देता है| इसके बाद remote FTP server से एक TCP connection बनाता है| server authorization के बाद connection को acknowledge करता है| इसके बाद user locate system से files को remote file system पर FTP command की सहायता से copy करता है| 
FTP file transfer करने के लिए दो parallel TCP connection को use करता है-
1. Control Connection
2. Data Connection
Control connection का कार्य control information भेजना है जैसे कि user identification,password,command to change remote directory और put व get file etc. तथा data connection का कार्य files को भेजने का होता है|

4.4 electronic mail in the internet: IMAP, POP,HTTP:-

IMAP:-

IMAP (internet mail access protocol) है| यह local server से email को access करने के लिए IMAP एक standard protocol है| IMAP 1986 में designed किया गया था जो एक remote server पर email को stored करने तथा remote access करने को allow करता है| IMAP एक client server protocol है जो email को received करता है तथा जो internet server द्वारा held किये हुए है| 
IMAP server पर mails को stored करके रखता है| यह real time में mailbox के contents को accessing and editing को allow करता है| आप अपने mail को multiple locations से access कर सकते है| आप server पर mail box or multiple folders को manipulate तथा create कर सकते है तथा msg को delete कर सकते है | तथा बिना download के header information को देख सकते है| 

POP:-

Post office protocol(POP) एक application layer internet standard protocol है| जिसका use mail server से email retrieve करने के लिए किया जाता है| most email application POP protocol का use करती है| यद्यपि कुछ application newer IMAP का use करती है| 
यहाँ POP के 2 version है-
1. POP2
2. POP3
POP का first version POP2 कहलाता है जो mid 80’s में standard है और इसमें msg को send करने के लिए SMTP की requirement होती है| 
POP का second version POP3 कहलाता है इसमें msg को send करने के लिए SMTP की requirement नहीं होती है| 
POP protocol में user computer पर outgoing emails को locally stored किया जाता है| अर्थात user द्वारा use किये गए computer से ही stored email को प्राप्त किया जाता है| 
POP remote mail box को उपयोग में लाने के लिए download and delete requirement को support करता है| जिसमे POP के द्वारा msg को server से download करने के बाद delete किया जा सकता है,पहले नहीं| यदि user किसी msg को delete नहीं करता तो वह msg next time mail checking के time पर पुनः download हो जायेगा| 
POP “offline” mail processing को support करता है जिसमे हम offline रहते हुए भी email को check कर सकता है,read कर सकते है| यह standard protocol अधिक popular email protocols में built किया गया है| 
POP3 email receive करने के लिए एक standard protocol का most recent version है| POP3 एक client server protocol है जो internet server पर रखे गए email को receive करता है तथा periodically server पर mailbox को check करता है तथा POP3 का use करके किसी भी mail को download करता है| 


HTTP:-

HTTP का अर्थ Hyper Text Transfer Protocol है। यह web पर primary technology protocol हैं, जो लिंकिंग और ब्राउज़िंग को संभव बनाता है। यह web server और web user के बीच communicate करने के लिए उपयोग की जाने वाली technology है।
यह protocol बड़े, multi- functionig, multi-input system का base है – जैसे world wide web।वेब जैसा कि हम जानते हैं कि यह बिना कम्युनिकेशन प्रोसेस के फंक्‍शन नहीं करेगा, क्योंकि लिंक ठीक से काम करने के लिए HTTP पर निर्भर करते है।
यह एक standard internet protocol है जो कि web browser जैसे MS internet explorer और web server जैसे MS internet information service(IIS) के बीच client server interaction process specify करता है | 
HTTP दो program द्वारा implement किया गया है एक client program तथा दूसरा server program client एवं server program अलग-अलग system पर run करता है| HTTP msg का structure client और server के बीच msg को exchange करता है| 
HTTP web client की webpage request को web server को देता है| server web page को client पर transfer करता है|

4.5 WWW:-

WWW का full form है World Wide Web. जिसको W3 या Web भी बोला जाता है. ये एक information space है जहाँपे HTML document और web resources को Uniform Resource Locators के जरिये Identify किया जाता है. जहाँ HTML document Hyper link के जरिए आपस में जुड़े रहते हैं. इन web Document को हम Internet के जरिए access करते हैं.
Information को Store करने का ये Central Hub है. WWW, एक primary tool है जिसके जरिये हम Internet को Access या communicate करतेहैं. web document को HTML (Hyper Text MarkUP Language) के जरिए लिखा जाता है. HTML document के अंदर image, audio, Video या text को Formatted वे में रखा जाता है.
ये एक बोहत बड़ा network हैं जहाँ Hypertext file और web pages आपसमें linked हैं. जहाँ billions of hypertext text document अलग अलग Servers और Computer में Store करके रखते हैं. जिनको web browser के जरिए Access किया जाता है.
इसको हम एक system भी बोल सकते हैं जहाँ Hyper text document आपस में inter linked हैं, इनको access करने के लिए Internet का इस्तेमालb किया जाता है.
इसको हम Virtual place बोल सकते हैं जहाँ दुनिया के सारे web pages, web server, Website को HTTP protocol के जरिए access किया जाता है
. WWW में web pages, web servers, URL, HYPERLINKS और HTTP का collection भी बोल सकते हो. इनके बारे में निचे दिया गया है.
ये resources और Users, जो HTTP के जरिए Internet से जुड़ते हैं उनका Medium है.
दुनिया में जितने भी websites और web pages internet में हैं उनके combination को वर्ल्ड वाइड वेब बोला जाता है.

4.1 principle of application layer protocol:-नेटवर्क एप्लिकेशन एक कंप्यूटर नेटवर्क के डीटेल हैं। हम अगर किसी भी उपयोगी अनुप्रयोगों के बारे में गर्भ धारण नहीं कर सकते, इसके लिए कोई आवश्यकता नहीं होगी उनका समर्थन करने के लिए नेटवर्किंग प्रोटोकॉल डिज़ाइन करें। लेकिन पिछले तीस वर्षों में, कई लोगों ने कई सरल और अद्भुत नेटवर्किंग तैयार की हैं अनुप्रयोगों। इन अनुप्रयोगों में क्लासिक पाठ-आधारित अनुप्रयोग शामिल हैं 1980 के दशक में यह लोकप्रिय हो गया, जिसमें कंप्यूटरों की रिमोट एक्सेस शामिल है, इलेक्ट्रॉनिक मेल, फ़ाइल स्थानांतरण, समाचार समूह और चैट। लेकिन वे भी शामिल हैं अभी हाल ही में वर्ल्ड वाइड जैसे मल्टीमीडिया अनुप्रयोगों की कल्पना की गई है वेब, इंटरनेट टेलीफोनी, वीडियो कॉन्फ्रेंसिंग, और ऑडियो और वीडियो पर मांग। हालांकि नेटवर्क एप्लिकेशन विविध हैं और कई परस्पर क्रिया करते हैं घटकों, सॉफ्टवेयर लगभग हमेशा उनके मूल में है। खंड से याद करें 1.2 कि एक नेटवर्क एप्लिकेशन का सॉफ्टवेयर दो या अधिक के बीच वितरित किया गया है एंड सिस्टम (यानी, होस्ट कंप्यूटर)। उदाहरण के लिए, वेब के साथ हैं सॉफ्टवेयर के दो टुकड़े जो एक दूसरे के साथ संवाद करते हैं: ब्राउज़र उपयोगकर्ता के मेजबान (पीसी, मैक, या वर्कस्टेशन) और वेब सर्वर में सॉफ्टवेयर वेब सर्वर में सॉफ्टवेयर। टेलनेट के साथ, फिर से दो टुकड़े हैं दो मेजबान में सॉफ्टवेयर: स्थानीय होस्ट में सॉफ्टवेयर और रिमोट में सॉफ्टवेयर मेज़बान। मल्टीपार्टी वीडियो कॉन्फ्रेंसिंग के साथ, प्रत्येक में एक सॉफ्टवेयर पीस है मेजबान जो सम्मेलन में भाग लेता है। ऑपरेटिंग सिस्टम के शब्दजाल में, यह वास्तव में सॉफ्टवेयर टुकड़े नहीं है (अर्थात, प्रोग्राम) जो संचार कर रहे हैं लेकिन सत्य प्रक्रियाओं में हैं संवाद। एक प्रक्रिया को एक प्रोग्राम के रूप में सोचा जा सकता है जो चल रहा है 
एक अंत प्रणाली के भीतर। जब संचार प्रक्रियाएं चल रही हों एक ही अंत प्रणाली, वे एक-दूसरे के साथ इंटरप्रोसेस का उपयोग करके संवाद करते हैं संचार। इंटरप्रोसेस संचार के लिए नियमों द्वारा नियंत्रित किया जाता है एंड सिस्टम का ऑपरेटिंग सिस्टम लेकिन इस किताब में हमें कोई दिलचस्पी नहीं है एक ही मेजबान पर प्रक्रिया कैसे संवाद करती है, लेकिन इसके बजाय कैसे विभिन्न अंत प्रणालियों पर चलने वाली प्रक्रियाएं (संभावित रूप से भिन्न के साथ) ऑपरेटिंग सिस्टम) संचार। दो अलग-अलग अंत प्रणालियों पर प्रक्रियाएं संदेशों का आदान-प्रदान करके एक दूसरे के साथ संवाद करें कंप्यूटर नेटवर्क। एक भेजने की प्रक्रिया बनाता है और संदेश भेजता है जाल; एक प्राप्त करने की प्रक्रिया इन संदेशों को प्राप्त करती है और संभवतः संदेश वापस भेजकर प्रतिक्रिया करता है,
नेटवर्किंग अनुप्रयोगों में अनुप्रयोग-परत प्रोटोकॉल होते हैं जो प्रारूप को परिभाषित करते हैं और संदेशों का क्रम प्रक्रियाओं के बीच आदान-प्रदान, साथ ही परिभाषित करता है संदेश के प्रसारण या प्राप्ति पर की जाने वाली क्रियाएं।एप्लिकेशन परत एक विशेष रूप से अच्छी जगह है प्रोटोकॉल। यह परिचित जमीन है। हम इनमें से कई से परिचित हैं ऐसे अनुप्रयोग जो प्रोटोकॉल पर निर्भर करते हैं, हम उनका अध्ययन करेंगे। यह हमें एक अच्छा देगा क्या प्रोटोकॉल के बारे में महसूस कर रहे हैं और हमें के कई के लिए मिलवा देंगे जब हम परिवहन, नेटवर्क, और डेटा-लिंक लेयर प्रोटोकॉल।


4.2 Doman name system (DNS):-Domain Name System को short मे DNS भी कहते है |DNS का काम Domain Name को IP Address मे convert करना होता है | जब हम किसी भी domain name को web browser मे टाइप करते है तो ये DNS उसको IP Address मे convert करता है | इसी प्रकार किसी भी Network मे किसी भी Computer एवं host name को भी ये IP Address मे कन्वर्ट करता है| यह इसलिए होता है क्योकि हम IP Address की तुलना मे Alphanumeric Name easily याद रख सकते है |एक उदहारण की लिए – जब हम http://www.example.com को browser मे type करते है तो DNS इसको 198.15.45.18 मे या इसी प्रकार की किसी valid IP Address मे चेंज कर देता है| अब आप खुद देखिए हमको http://WWW.example.com याद करना कितना आसान है जबकि IP address याद करना कितना मुश्किल| 
अब हम जानते है की Public IP Address के case मे अन गिनत DNS और IP Address हो सकते है तो केवल एक सर्वर सभी की information को save करके नहीं रख सकता, इसी वजह से जब query एक सर्वर के पास आती है एवं उस सर्वर को IP Address नहीं पता होता या वो उस query को resolve नहीं कर पता तो वो query को उससे connected दूसरे सर्वर के पास भेज देता है| यह Pass on तब तक चलता है जब तक query resolve नहीं हो जाती एवं IP Address नहीं मिल जाता|

Working of DNS:-

1. सबसे पहले user किसी website का URL web browser के address bar में लिखता है और enter press करता है| 
2. Browser सबसे पहले अपने cache को check करता है कि क्या उसके पास इस domain name का IP address पहले से मौजूद है| जब भी हम किसी website को visit करते है तो उसका IP address browser अपने cache में कुछ दिनों के लिए store कर लेता है| ऐसा इसलिए करता है कि ताकि same process को बार बार न दोहराया जाये और server पर load भी न पड़े| 
3. यदि browser को cache में ही IP address मिल जाता है तो user को उस IP address से connect कर दिया जाता है और website load हो जाती है| 
4. यदि browser को IP address अपने cache में नहीं मिलता है तो browser ये request OS को transfer करता है| OS इस request को resolver को transfer करते है| 
5. Resolver इस domain name के IP address के लिए खुद के cache को check करता है| यदि IP address मिल जाता है तो resolver इसे OS को और OS web browser को transfer कर देता है| 
6. यदि resolver को IP address नहीं मिलता तो वह इस request को root server तक पहुँचाता है| एक resolver आपका ISP (internet service provider) होता है| सभी resolver को root server के बारे में जानकारी होती है| 
7. Root server इस request को top level domain server को pass कर देता है| Top level domain server को com,gov,edu,org,in आदि servers के बारे में जानकारी होती है| इसलिए वह इस request को appropriate top level domain server को pass कर देता है| 
8. Top level domain server इस request को authoritative name server को pass कर देता है अभी तक की process में com server को locate किया जा चूका है,अब actual name server को locate किया जायेगा| जब भी कोई domain register की जाती है तो domain register उस domain name को store कर और top level domain servers के साथ इस information को share करता है| इसलिए top level domain server को पता रहता है कि actual name server कहाँ है| 
9. Authoritative name server request की गयी domain name का IP address provide करता है| Resolver इस IP address को अपने cache में store कर लेता है| साथ ही OS भी इस IP address को further request को serve करने के लिए अपने cache में store कर लेता है| 
10. आखिर में resolver IP address के base पर website को browser में load करता है| 


4.3 The file transfer protocol (FTP):-FTP की full form File Transfer Protocol होती है| यह एक application layer protocol है जो एक local file system से remote file system के मध्य files transfer करता है| इसके लिए एक FTP client व FTP server की आवश्यकता होती है| 
FTP user को इस योग्य बनाता है कि वह remote system में login कर सके| जब user login कर चूका होता है तब वह उस system पर directory की जाँच कर सकता है, files में changes कर सकता है,commands को execute कर सकता है और साथ ही कुछ अन्य कार्य भी कर सकता है| 
FTP एक client-server प्रोटोकॉल है इसलिए यह client और server के मध्य डेटा ट्रान्सफर करने के लिए दो TCP connections स्थापित करता है। FTP द्वारा एक TCP connection का प्रयोग conservation को नियंत्रित करने के लिए किया जाता है तथा दूसरे TCP connection का प्रयोग डेटा files को ट्रान्सफर करने के लिए किया जाता है।
सर्वप्रथम user FTP user agent की सहायता से Remote System पर login करता है जिसके लिए आवश्यक user name व password देता है| इसके बाद remote FTP server से एक TCP connection बनाता है| server authorization के बाद connection को acknowledge करता है| इसके बाद user locate system से files को remote file system पर FTP command की सहायता से copy करता है| 
FTP file transfer करने के लिए दो parallel TCP connection को use करता है-
1. Control Connection
2. Data Connection
Control connection का कार्य control information भेजना है जैसे कि user identification,password,command to change remote directory और put व get file etc. तथा data connection का कार्य files को भेजने का होता है|

4.4 electronic mail in the internet: IMAP, POP,HTTP:-

IMAP:-

IMAP (internet mail access protocol) है| यह local server से email को access करने के लिए IMAP एक standard protocol है| IMAP 1986 में designed किया गया था जो एक remote server पर email को stored करने तथा remote access करने को allow करता है| IMAP एक client server protocol है जो email को received करता है तथा जो internet server द्वारा held किये हुए है| 
IMAP server पर mails को stored करके रखता है| यह real time में mailbox के contents को accessing and editing को allow करता है| आप अपने mail को multiple locations से access कर सकते है| आप server पर mail box or multiple folders को manipulate तथा create कर सकते है तथा msg को delete कर सकते है | तथा बिना download के header information को देख सकते है| 

POP:-

Post office protocol(POP) एक application layer internet standard protocol है| जिसका use mail server से email retrieve करने के लिए किया जाता है| most email application POP protocol का use करती है| यद्यपि कुछ application newer IMAP का use करती है| 
यहाँ POP के 2 version है-
1. POP2
2. POP3
POP का first version POP2 कहलाता है जो mid 80’s में standard है और इसमें msg को send करने के लिए SMTP की requirement होती है| 
POP का second version POP3 कहलाता है इसमें msg को send करने के लिए SMTP की requirement नहीं होती है| 
POP protocol में user computer पर outgoing emails को locally stored किया जाता है| अर्थात user द्वारा use किये गए computer से ही stored email को प्राप्त किया जाता है| 
POP remote mail box को उपयोग में लाने के लिए download and delete requirement को support करता है| जिसमे POP के द्वारा msg को server से download करने के बाद delete किया जा सकता है,पहले नहीं| यदि user किसी msg को delete नहीं करता तो वह msg next time mail checking के time पर पुनः download हो जायेगा| 
POP “offline” mail processing को support करता है जिसमे हम offline रहते हुए भी email को check कर सकता है,read कर सकते है| यह standard protocol अधिक popular email protocols में built किया गया है| 
POP3 email receive करने के लिए एक standard protocol का most recent version है| POP3 एक client server protocol है जो internet server पर रखे गए email को receive करता है तथा periodically server पर mailbox को check करता है तथा POP3 का use करके किसी भी mail को download करता है| 


HTTP:-

HTTP का अर्थ Hyper Text Transfer Protocol है। यह web पर primary technology protocol हैं, जो लिंकिंग और ब्राउज़िंग को संभव बनाता है। यह web server और web user के बीच communicate करने के लिए उपयोग की जाने वाली technology है।
यह protocol बड़े, multi- functionig, multi-input system का base है – जैसे world wide web।वेब जैसा कि हम जानते हैं कि यह बिना कम्युनिकेशन प्रोसेस के फंक्‍शन नहीं करेगा, क्योंकि लिंक ठीक से काम करने के लिए HTTP पर निर्भर करते है।
यह एक standard internet protocol है जो कि web browser जैसे MS internet explorer और web server जैसे MS internet information service(IIS) के बीच client server interaction process specify करता है | 
HTTP दो program द्वारा implement किया गया है एक client program तथा दूसरा server program client एवं server program अलग-अलग system पर run करता है| HTTP msg का structure client और server के बीच msg को exchange करता है| 
HTTP web client की webpage request को web server को देता है| server web page को client पर transfer करता है|

4.6 Network management:-

SNMP एक network management protocol है| यह application layer पर कार्य करता है| इसे 1988 में create किया गया था| SNMP को RFC 1065 में define किया गया है | SNMP के द्वारा किसी device का status real time में प्राप्त किया जा सकता है| इसके द्वारा device को configure भी किया जा सकता है| 
मान लीजिये किसी company में 300 network device है| जैसे कि routers,switches,hubs,printers आदि| हर रोज company को इनका status देखना होता है| ऐसे में इतने devices को check करना बहुत मुश्किल काम है| ऐसी situation में SNMP बहुत उपयोगी होता है| SNMP द्वारा devices का current status प्राप्त किया जा सकता है| इससे devices तक जाने की आवश्यकता नहीं होती है|
