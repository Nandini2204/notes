7.2 Finding Files:-unix के पुरे फाइल hierarchy में किसी फाइल को search करना हो तो हम find command use करते है | find command के द्वारा हम proper फाइल को search कर सकते है साथ ही उस पर कोई action भी perform कर सकते है|
find command line थोड़ी कठिन है ओर इसके लिए निम्न syntax है 
find < path_list>< selection_criteria>enter
Operation of find command:-
1. Find command specified path list की directory में से सभी फाइल्स को recursively examine करना है |
2. ओर तब प्रत्येक फाइल की criteria को एक से अधिक पर selection करके check करता है |
3. अंत में उन selected फाइल्स पर action perform करता है |
हम किसी भी फाइल को निम्न parameters का use करके भी find कर सकते है |
a) type
b) owner
c) time
d) etc
syntax :-
find< starting directory>< selection_criteria>< action>enter
< selection criteria>
1. -atime:- इसका तात्पर्य है की access time के आधार पर find करता है |
example :- -atime 2{last 2 days accessed}
2.-newer:- फाइल्स की जगह फाइल्स का नाम देगे तब उस फाइल के जो भी new फाइल बनी होगी वह terminal पर show होगी|
3. –size 
4. –print etc.
5. –permission
6.-no user
7.-system type

7.5 Compressing files:-The most common programs used to compress files in Unix-like systems are:
1. gzip
2. bzip2
Tar Command:-
Tar command मुख्य रूप से आवश्यक फ़ाइलों या directory को एक संग्रह में bundle करने के लिए उपयोग किया जाता है। उपयोग किए गए option के आधार पर हम Tar command की content को निकालने या सूचीबद्ध करने के लिए Tar command का भी उपयोग कर सकते हैं। फ़ाइलों को archive और compress करने के लिए tar command का उपयोग करने के लिए syntax निम्नानुसार है:
tar [-] c|x|t f tarfile [pattern]
Some of the most common options used along with the tar command are as follows:
c – Create a tar archive
x – Extract files from the archive
t – Display the table of contents (list)
v – Be verbose
z – Use compression
f – Used to specify the file to use

Gzip and Gunzip Command:- unix में फ़ाइलों को कैसे archive और compress किया जाए, इस पर हमारे tutorial का अगला भाग compressing फ़ाइलों पर focues है। फ़ाइलों को compress करने और मेमोरी स्पेस को बचाने के लिए, हम gzip कमांड का उपयोग करते हैं। फ़ाइलों को uncompress करने के लिए, हम gunzip कमांड का उपयोग करते हैं।

7.6 Backing up files(tar, cpio):-Back up एक प्रमुख कार्य है जो एक system adminstrator करता है। शुरुआती beginners के लिए यहां कुछ hint दिए गए हैं। आपको उस tape device का नाम बदलना होगा जिसका आप उपयोग कर रहे हैं। कुछ स्क्रिप्ट यहाँ देखें। यदि आप जल्दी में हैं और ufsdump और ufsrestore का उपयोग करना चाहते हैं, तो नीचे उदाहरण देखें।
Using cpio:-
cpio copies files to and from archives.
Syntax:-
cpio -o [ a ] [ c ] [ v ] [ B | C Value ] < FileName >Output
cpio -i [ b ] [ c ] [ d ] [ f ] [ m ] [ r ] [ s ] [ t ] [ u ] [ v ] [ S ] [ 6 ] [ B | C Value ] [ Pattern … ] < Input
cpio -p [ a ] [ d ] [ l ] [ m ] [ u ] [ v ] Directory < FileName
cpio -o Command:-
Cpio -o कमांड मानक इनपुट से फ़ाइल पथ नामों को पढ़ता है और इन फ़ाइलों को मानक नाम और स्थिति जानकारी के साथ मानक आउटपुट में कॉपी करता है। कई विशिष्ट रूप से लिंक की गई फ़ाइलों से बने cpio कमांड पथ के नाम देने से बचें, क्योंकि उनमें से किसी को ट्रैक करने के लिए पर्याप्त मेमोरी नहीं हो सकती है और लिंकिंग जानकारी को खो देगा।
cpio -i Command:-
Cpio -i कमांड, cpio -o कमांड द्वारा बनाई गई मानक इनपुट से एक आर्काइव फ़ाइल को पढ़ता है और इससे उन फाइलों को कॉपी करता है, जो पैटर्न पैरामीटर से मेल खाती हैं।
cpio -p Command:- Cpio -p कमांड मानक इनपुट से फ़ाइल पथ नामों को पढ़ता है और इन फ़ाइलों को निर्देशिका पैरामीटर द्वारा नामित निर्देशिका में कॉपी करता है। निर्दिष्ट निर्देशिका पहले से मौजूद होनी चाहिए। यदि इन पथ नामों में निर्देशिका नाम शामिल हैं जो पहले से मौजूद नहीं हैं, तो आपको निर्दिष्ट निर्देशिका बनाने के लिए d ध्वज का उपयोग करना चाहिए।
आप विशेष फ़ाइलों को केवल तभी कॉपी कर सकते हैं जब आपके पास रूट उपयोगकर्ता authority हो। 
Parameters:- < Directory>:- Directory specifies करता है।
< FileName> :-input के रूप में उपयोग करने के लिए cpio command के लिए फ़ाइल नामों की एक सूची specifies करता है।
< Output>:- output device को diskette या फ़ाइल के रूप में specifies करता है। 
< Input>:- input device को specifies करता है (जहां इनपुट cpio -o कमांड द्वारा निर्मित आउटपुट फाइल है)। 
< Pattern>:- pattern को specifies करता है |
Tar command:-
unix tar’का मतलब tape archive से है, इसका इस्तेमाल archive बनाने और archived files को निकालने के लिए किया जाता है। unix में tar command एक महत्वपूर्ण command है जो unix में archiving functionality provide करता है। हम compressed या uncommressed files को बनाने और उन्हें बनाए रखने और संशोधित करने के लिए unix tar command का उपयोग किया जाता है |
Syntax:
tar [options] [archive-file] [file or directory to be archived]
Options:
-c : Creates Archive
-x : Extract the archive
-f : creates archive with given filename
-t : displays or lists files in archived file
-u : archives and adds to an existing archive file
-v : Displays Verbose Information
-A : Concatenates the archive files
-z : zip, tells tar command that create tar file using gzip
-j : filter archive tar file using tbzip
-W : Verify a archive file
-r : update or add file or directory in already existed .tar file
3. Extracting files from Archive using option -xvf : This command extracts files from Archives.
$ tar xvf file.tar
Output :-
os2.c
os3.c
os4.c

7.8 Understanding /etc/passwd, /etc/shadow, /etc/inittab:-/etc/passwd file format:-unix / etc / passwd फ़ाइल में अधिकांश account feature को store करता है। यह एक text-based फ़ाइल है जिसमें प्रत्येक entry के लिए सात field हैं। प्रत्येक entry एक user नाम से शुरू होती है और कॉलन द्वारा अलग किए गए field के एक set के साथ जारी रहती है (:)। यहाँ / etc / passwd फ़ाइल में प्रविष्टि का syntax है:
username:password:UID:GID:comment:home directory:default shell
Here is a description of each field:
username – the name of the user.
Password- यह field traditionally रूप से passwoed के लिए आरक्षित किया गया है। अधिकांश unix वितरण, हालांकि, password को / etc / shadow में store करते हैं। यह field आमतौर पर x पर सेट होती है, जिसका अर्थ है कि shadow password उपयोग में हैं। यदि field asterisk (*) पर सेट है, तो इसका मतलब है कि खाते में कोई पासवर्ड configure नहीं है। यदि field exclamation mark (!) पर सेट है, तो खाता लॉक हो गया है।
UID:- user identifier. It is a unique number representing the user.
GID:- the default (primary) login group ID. Users have only one primary group and that group is the group owner for all new files the user creates.
Comment:- this field contain information about the user. Usually holds the user’s full name.
Home directory:- the absolute path to the user’s home directory.
Default shell:- the user’s default shell.

/etc/shadow file:- / Etc / shadow फ़ाइल user के account से जुड़े अतिरिक्त गुणों के साथ user के account के लिए एन्क्रिप्टेड प्रारूप (पासवर्ड के हैश की तरह) में वास्तविक पासवर्ड store करता है। असल में, यह सुरक्षित user account की information store करता है। सभी field को एक colon (:) symbol द्वारा अलग किया जाता है। इसमें / etc / passwd फ़ाइल में listed प्रत्येक user के लिए प्रति पंक्ति एक प्रविष्टि है।

Username : It is your login name. Password: यह आपका encrypted पासवर्ड है। पासवर्ड न्यूनतम 8-12 character का होना चाहिए, जिसमें विशेष character, अंक, कम केस अक्षर और अधिक शामिल हैं। आमतौर पर पासवर्ड प्रारूप $ id$salt$hashad पर सेट किया जाता है, $ id GNU/UNIX पर उपयोग किया जाने वाला algorithm निम्नानुसार है:
1.$1$ is MD5
2.$2a$ is Blowfish
3.$2y$ is Blowfish
4.$5$ is SHA-256
5.$6$ is SHA-512
3.Last password change (lastchanged) : Days since Jan 1, 1970 that password was last changed
/etc/inittab:-/ Etc / inittab फ़ाइल unix में System V (SysV) initialization system द्वारा उपयोग की जाने वाली कॉन्फ़िगरेशन फ़ाइल है। यह फ़ाइल init प्रक्रिया के लिए तीन वस्तुओं को परिभाषित करती है:
the default runlevel
what processes to start, monitor, and restart if they terminate
what actions to take when the system enters a new runlevel
एक बार आपके रनवे के लिए / etc / inittab की सभी entres execute हो जाने के बाद, boot प्रक्रिया पूरी हो जाती है, और आप लॉग इन कर सकते हैं।
इनटैब फ़ाइल की प्रत्येक पंक्ति में चार कोलोन-सीमांकित फ़ील्ड होते हैं:
id:runlevels:action:process


Here is a description of these fields:- id (identification code) – एक से four character का एक ssequance होता है जो इसके कार्य को पहचानता है |
runlevels – उस run level को listed करता है जिस पर यह enter लागू होती है।
action – इस field में specific code प्रक्रिया का इलाज करने के तरीके के बारे में बताते हैं। संभावित मानों में शामिल हैं: initdefault, sysinit, boot, bootwait, Wait, और respawn।
process – execute करने के लिए command या script को परिभाषित करता है।
