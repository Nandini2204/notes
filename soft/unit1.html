1.1 Software Engineering evolution and impact :- Software Evolution एक term है, जो शुरू में सॉफ्टवेयर को develop करने की process को संदर्भित (refer) करता है, फिर समय-समय पर इसे विभिन्न कारणों से update करता रहता है, यानी new features को जोड़ना या अप्रचलित कार्यों (obsolete functionalities)को दूर करना आदि।Evolution process includes change analysis of fundamental activities, release planning , system implementation and release a system to customer |
इन changes का एक impact cost तक पहुँच जाता है यह देखने के लिए कि changes से कितना system affect होता है और changes को लागू करने में कितना खर्च हो सकता है। यदि proposed changes accepted किए जाते हैं, तो सॉफ़्टवेयर सिस्टम release की योजना बनाई जाती है। release planning के दौरान, सभी proposed changes(fault repair, adaptation and new functionalities)पर विचार किया जाता है।

1.3 Program and software product :-

Program:-

एक सॉफ्टवेयर प्रोग्राम को आमतौर पर instructions के एक सेट या modules या procedures के एक सेट के रूप में define किया जाता है, जो एक निश्चित प्रकार के computer operation के लिए अनुमति देता है। इस शब्द का उपयोग अक्सर “सॉफ़्टवेयर एप्लिकेशन” और “सॉफ़्टवेयर product” जैसे शब्दों के साथ किया जाता है।
इन्हे निम्न प्रकार से डिफाइन किया जाता है
Size:- वे आम तौर पर छोटे sizeके होते हैं। codeकी lines 100-2000 line code या कुछ और हो सकती हैं।
Documentation:- Documentation में कोई डॉक्यूमेंटेशन या लॉक नहीं है।
USER:-Developer himself is sole user.
Developer:- Single developer or may be 2 developer make a program.
USER Interface:-There is no or lock proper user interface.
Development:- Unplanned,Not systematic I,e AD-HOC development.
Functionality:-It provide more functionality as they are big in size(lines of code)

Software Product:-

सरल शब्दों में, “Software is a collection of separate programs”। मूल रूप से एक software system में आमतौर पर कई अलग-अलग प्रोग्राम, configuration files (इन प्रोग्राम्स को सेट करने के उद्देश्य से), system documentaion (सिस्टम की संरचना का वर्णन करने के उद्देश्य से) और user documentation शामिल होते हैं|

Fundamental Type of Software Product:-

Software engineers का संबंध software product को develop करने से है, अर्थात, Software जिसे customer को बेचा जा सकता है। Software product के दो मूलभूत प्रकार हैं;
Generic products:- Generic software product stand-alone system हैं जो एक development organization द्वारा produced किए जाते हैं और open market में किसी भी customer को बेचे जाते हैं जो उन्हें खरीदने में सक्षम हैं। Generic product के example में personal computer (पीसी) के लिए software शामिल हैं जैसे data base mangement, word processor environment, art, drawing और animation packages और projrct management tool शामिल है |


Customized (or bespoke) products:- customized products software एक system है जो एक विशेष customer द्वारा commissioned किया जाता है। एक software contractor / vendor customer की आवश्यकता के अनुसार विशेष रूप से उस customer के लिए software develop करता है। software system के customized products के उदाहरणों में इलेक्ट्रॉनिक devices के लिए control system, bnking software, voice communication software system, एक विशेष business process और air traffic contorl system के समर्थन के लिए लिखी गई system शामिल हैं।

1.4 Software Characteristics:-एक अच्छे सॉफ्टवेयर के निम्नलिखित गुणधर्म होते है:-
1:-Robustness:- सॉफ्टवेयर में robustness होनी चाहिए जिससे कि सॉफ्टवेयर में defects तथा failure को detect किया जा सकें।
2:-Security:-सॉफ्टवेयर में security बेहतर होनी चाहिए जिससे कि सॉफ्टवेयर सम्पूर्ण सुरक्षा बनी रहें और वह threats से सुरक्षित रहें।
3:-Reliability:-सॉफ्टवेयर reliable होना चाहिए अर्थात इसमें कोई defects नही होने चाहिए।
4:-Flexibility:-सॉफ्टवेयर flexible होना चाहिए अर्थात अगर सॉफ्टवेयर में कुछ बदलाव करने है तो वह आसानी से किये जा सकें।
5:-Testability:-सॉफ्टवेयर की टेस्टिंग आसानी से की जा सकें।
6:-Platform independent:-सॉफ्टवेयर platform independent होना चाहिए अर्थात वह किसी भी सिस्टम में run हो सकें।

1.6 Emergence of Software engineering:-

More focus on hardware:-

कंप्यूटिंग के शुरुआती दिनों में primary concern हार्डवेयर के निर्माण या प्राप्त करने के साथ थी। यह सॉफ्टवेयर काम नहीं करता था, लोगों ने सोचा, यह काम करने तक इसे बदलना काफी आसान होगा।

Software cost consideration:-

सॉफ्टवेयर की लागत में हार्डवेयर की लागत की इतनी छोटी कार्रवाई होती है कि कोई भी इसे बहुत महत्वपूर्ण नहीं मानता है। लोगों का समय मशीन को बचाने के लिए रखा गया था।

Meeing People’s Expectations:-

जब कंप्यूटिंग matured हो जाती है, तो program अधिक complex और बड़े हो जाते हैं। एक team प्रयास यदि program के लिए आवश्यक है कि नियमित रूप से specified, written, operated और बनाए रखा जाए, तो लोगों की उम्मीदें पूरी हो जाएंगी।

Exceeded code:-

चूंकि program बड़े और complex हो जाते हैं और सभी अनुमानों को पार करने के लिए program और उनकी लागतों को लिखने के लिए आवश्यक समय हो जाता है।

Time-Development:-

programmers को पूर्ण होने की अपेक्षा सप्ताह, मंथ या वर्ष अधिक लगने लगे। client को बार-बार चालू करने से सिस्टम सही ढंग से काम नहीं कर पाता है, इससे पहले कि प्रोग्राम को आर्गनाइज किया जा सके, पैसा या समय खत्म हो जाता।
हमारे मिलिट्री, सेवरल मिलियन डॉलर प्रोजेक्ट को खत्म कर दिया गया था क्योंकि यह नेवर को ठीक से काम करने के लिए मोड दे सकता था।

Quality Concern:-

चूंकि कंप्यूटर और उनके कार्यक्रमों का उपयोग जीवन की निगरानी जैसे अधिक महत्वपूर्ण कार्यों के लिए किया जाता था।

Change-Expensive:-

एक complex program के भीतर एक बदलाव करना बहुत महंगा हो गया। यह एक program फेंकने के लिए program से थोड़ा अलग काम करने की अपेक्षा करना आसान था।

Fast and Rapid change in hardware and software:-

जैसे कि ट्यूब को ट्रांसस्टिस्टर -> integrated circuits द्वारा बदल दिया गया।
हार्डवेयर की लागत जो कम हो गई थी। उसे कम समय मिल रहा था और सॉफ़्टवेयर को विकसित करने की लागत हार्डवेयर की तुलना में अब इतनी कम नहीं थी, कि उन्हें अनदेखा किया जा सके।

Wrong Analysis of client Requirement:-

program अक्सर सोचा गया था कि वे ग्राहक की आवश्यकता को समझ गए थे। एक बार program लिखे जाने के बाद, client ने असंतोष व्यक्त करना शुरू कर दिया।

No Satisfactory System:-

दुर्भाग्य से, कई वर्षों पहले तक प्रतिनिधित्व का कोई अच्छा तरीका मौजूद नहीं था, जो संतोषजनक तरीके से complex systems का वर्णन करते हैं, जैसे कि आज उन्हें हटा दिया गया है।
